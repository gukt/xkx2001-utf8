# 25 — 场景内容：野外 + 官道 + 渡口（遭遇战斗 + 骑乘通行 + 渡船）

**What to build:** 落地 [10 号票](../../mvp-scope/issues/10-mvp-scenes-selection.md) 定稿的野外/官道/水陆交通场景内容（spec 用户故事 51、57–60、66、块 H）：一段扬州<->少林的官道 + 一片野外遭遇区（扬州郊外森林概念房间）+ 官道沿途一处渡口/渡船。野外至少一个房间放置会主动攻击玩家的 NPC（挂 `Behaviors` `kind="aggro"`，如山贼/野兽，对应 19 号票机制真实验收场），击败后掉落战利品、声明 `respawn` 支撑长期游玩（18 号票机制真实验收场）。官道与野外允许骑乘通行（部分路段声明较高 `Terrain.cost`，示范"不是所有坐骑都能走遍天下"，15 号票机制真实验收场）。渡口挂 `Ferry`（09 号票机制真实验收场），`look` 能看到渡船状态提示。房间键使用 `wild_*`/`road_*`/`ferry_*` 前缀。

**Blocked by:** 09（`Ferry`），15（`Terrain`+坐骑限制），18（NPC 死亡/战利品/重生），19（aggro 行为）。

**Status:** ready-for-agent

- [ ] 官道至少数个房间连接扬州与少林方向（与 22/24 号票的出口方向协调一致）；野外至少 2–3 个房间构成遭遇区。
- [ ] 野外至少一个 NPC 模板挂 `aggro` behavior + 有意义的 `Vitals`/招式配置（能被玩家打败，也可能打败低等级玩家——不要求平衡性调优，只要求机制真实可玩）；声明 `respawn=True` 与合理的战利品配置（金钱/物品）。
- [ ] 官道/野外至少一段房间声明较高 `Terrain.cost`，配合场景内容体现"某些坐骑走不过去，需下马步行"（需要在场景里能通过 `ride` 一匹低 `ability` 坐骑真实触发这条限制，作为验收路径之一）。
- [ ] 渡口两岸房间挂 `Ferry`（互相 `far_bank` 指向），`look` 能看到渡船位置/预计到达提示；渡船到岸时可 `go` 过河，不在时给出"渡船不在对岸"一类提示。
- [ ] 房间键统一 `wild_*`/`road_*`/`ferry_*` 前缀；官道两端分别连接 22/24 号票已建的扬州/少林出口。
- [ ] 端到端剧本片段测试：进入野外遭遇区 -> 被动或主动触发交战 -> 战胜后获得战利品/经验 -> 沿官道骑马前进（若坐骑不够格则先体验拒绝再 `unride` 步行）-> 抵达渡口等船过河。
- [ ] 现有测试全绿不回归；`just verify-*` 一键矩阵全绿。
