# 21 — 场景内容：华山村（新手教程）

**What to build:** 落地 [10 号票](../../mvp-scope/issues/10-mvp-scenes-selection.md) 定稿的新手出生点场景内容（spec 用户故事 61–62、块 H）：华山村作为中立新手村（不绑定后续门派选择），包含一位教程向导 NPC（通过 `ask` 对话讲解基础命令：移动/查看/拾取/战斗，并给出去向指引——去扬州闯荡还是投奔某个门派）+ 一次低风险战斗教学（一个不会反杀的稻草人/教学木桩 NPC，只挂 `Vitals`，不挂 `Behaviors`/`AIController`，玩家 `attack` 后能看到完整的战斗播报流程）。建议把教学木桩所在房间标记 `NoDeathZone`（06 号票组件），具体、真实地在场景内容里练一次"擂台/切磋安全区"这条机制，而不是只停留在单元测试里。房间键使用 `huashan_*` 前缀（为 26 号票的六分区互联铺路）。

**Blocked by:** 06（`NoDeathZone`），12（`attack` 命令 + 真实战斗播报）。

**Status:** ready-for-agent

- [ ] 华山村至少含出生房间 + 教程向导所在房间 + 教学木桩所在房间（房间数量与摆设细节由实现阶段按体验需要决定，无需照搬原版）。
- [ ] 教程向导 NPC：`ask <向导> about <话题>` 覆盖移动/查看/拾取/战斗四个基础话题 + 一个"去哪"话题给出扬州/门派两个方向指引（复用 M1 已有的 `Inquiry` 组件，不新增机制）。
- [ ] 教学木桩 NPC：挂 `Vitals`（有意义的 HP 数值，能被打倒或至少能承受多次攻击展示完整播报），不挂 `Behaviors`/`AIController`（不会反击）；`attack 木桩` 后能看到命中/伤害/剩余气血等完整战斗播报（复用 12 号票机制）。
- [ ] 教学木桩房间挂 `NoDeathZone`（若教学设计确实存在"玩家气血也可能被消耗至归零"的路径——若教学木桩完全不反击则玩家气血不会归零，`NoDeathZone` 更多是为未来"教学切磋"场景做示范，本条验收接受"即使当前设计下用不上也要挂上并有一条测试证明其生效"）。
- [ ] 端到端剧本片段测试（可先是本票范围内的小型脚本测试，26 号票会把它接进完整六分区剧本）：出生 -> `ask 向导 about 战斗` -> `attack 木桩` -> 收到完整播报。
- [ ] 房间键统一 `huashan_*` 前缀。
- [ ] 现有测试全绿不回归；`just verify-*` 系列一键矩阵全绿。
