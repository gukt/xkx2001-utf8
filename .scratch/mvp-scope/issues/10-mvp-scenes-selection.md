## Question

基于 [09-subsystem-classification-confirm](09-subsystem-classification-confirm.md) 的最终子系统归类结果(尤其 MVP 必做项),挑选武侠题材包 MVP 具体要做哪几个典型场景/房间——要求覆盖面最大、代表性最强的最小集合(例如起始房间、一个门派场景、一个城镇场景等,具体待定)。

## Answer

2026-07-18,基于 [09](09-subsystem-classification-confirm.md) 的最终归类结果,`/grilling` 方式确认。用户明确拍板的实质范围(逐条对话确认):门派选**少林**(而非最初提案的华山);城镇**扬州要做得丰富一些**(老玩家熟悉,不只是最小子集);额外要求**野外 + 官道(区域间道路) + 水陆交通 + 坐骑系统都要联通**——这一条直接触发 [08 号票](08-subsystem-classification-research.md) 里坐骑与交通系统(38)的归类从"现代化改造 + MVP 不做"改判为"MVP 必做"(见 08 号票档位分布最新版与 09 号票末尾追加说明)。

具体房间/地标级别的细节(用户对后续两轮更细的确认题选择跳过,按"框架/流程层面小决定 agent 自行拍板"处理,记录理由供后续 `/to-spec` 阶段参考,不是不可改的定论):

1. **新手出生点** = 华山村(`/d/village`)——保留作中立新手村,不强制绑定后续门派选择,给玩家在村内学基础命令、简单战斗教程,再决定去哪个门派或直接闯荡扬州。
2. **城镇** = 扬州(`/d/city`)丰富子集——中央广场(guangchang,枢纽+动态事件位)+ 四条大街各至少一段(beidajie/nandajie/dongdajie/xidajie)+ 四座城门(beimen/nanmen/dongmen/ximen)+ 客栈(kedian)+ 钱庄(qianzhuang，练手基础金钱系统)+ 打铁铺(datiepu)+ 镖局(biaoju)+ 武庙(wumiao)+ 茶馆(chaguan)。覆盖:多房间导航、NPC 商店交易、基础货币流通、老玩家地标记忆点。
3. **门派** = 少林寺(`/d/shaolin`)——山门(shanmen,严格身份/性别/武器权限校验,示范 `valid_leave` 边界控制)+ 广场(guangchang)+ 达摩院/藏经阁(剧情向房间)+ 武场(wuchang,技能学习教学位)+ 武僧/知客僧 NPC(教学对话 + 门派 NPC-AI 行为)。
4. **野外** = 扬州郊外(`forest` 概念房间,新引擎里作为扬州↔少林官道沿途的战斗/遭遇验证区),用于验证 NPC 随机刷新与野外战斗,不需要照搬原版具体森林房间数量。
5. **官道** = 一条扬州↔野外↔少林的跨区域连接道路,参考原版"终南山道路连接峨嵋"的跨区域 exit 模式,验证世界连接图/跨区域出口机制。
6. **水陆交通** = 官道沿途一个渡口/渡船实例(继承 FERRY 基类),跨过一条河,验证特殊房间类型(渡口/渡船)与运行时动态出口注入。
7. **坐骑** = 官道与野外沿途允许骑乘通行(消耗坐骑体力换移动效率),验证坐骑机制与"运行时动态出口注入解决地理断点"的设计思路。

这份场景清单覆盖的 MVP 必做子系统:标准命令(29)、房间继承(26)、数据存储(34)、角色(5)、命令调度(8)、金钱基础部分(17)、门派武学框架(30,虽标"现代化改造"但少林是 MVP 题材包要落地的具体内容)、技能(32)/状态(31)/战斗(7,依赖 02 号票暂定倾向)、死亡与轮回(39)、坐骑与交通(38,本次改判)、NPC-AI(41)。
