# Post-MVP backlog（MVP 里程碑 M0–M4 之后）

> 本文件记录**已确认要做、但不进当前 M0–M4** 的后续能力。不拆实施票；到排期时再开里程碑（如 M5）并 `/to-spec`。
> 决策依据：[ADR-0006](../../docs/adr/0006-no-engine-editor-board-post-mvp-creator-platform.md)。

## 候选里程碑（待排期）

### Web 创作者一站式平台（建议作 M5 或独立产品里程碑）

独立于游戏引擎进程的创作者产品，统一管理：

- 新题材 / 内容包的创作与资源管理
- 验证（复用引擎侧包校验契约 / CLI，而非在引擎内嵌 UI）
- 上架 / 下架
- 分成提现
- 数据统计
- 账户体系
- AI Agent 创作辅助（生成 / 改稿 / 评审编排）

**引擎侧只保留**：内容包格式与加载、可复用校验、运行时安全护栏、题材包元数据接缝（对齐 [06](issues/06-scaling-commercialization-support-points.md)）。不在 `mud_engine` 内建编辑器或 Agent 编排。

### 留言板 / 异步公告（多人世界互动阶段再评估）

LPC `bboard` / `F_EDIT` 式房间留言、文选收编（`editord.c`）**不做进引擎 MVP**。现代游戏少见该类机制；若未来做多人世界异步社交，再单独评估形态（不必复刻 MUD 公告板）。实时频道等仍属 MVP 必做（子系统 4），与本条无关。

| 留言板 / 公告板（`bboard` 等） | 不实现；列入上方「多人互动阶段再评估」 |
