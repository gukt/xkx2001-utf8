# M1 物品命令手测 / 一键矩阵

> 分支：`verify/m1-items`  
> 范围：物品 C1–C7 + `put` + `get`（`take` 别名）/`drop <数量>`/`get all`/`drop all`  
> 不含：NPC、Nature、完整 `eat`/`wield`/`give`

## 推荐：一键矩阵（免手动敲命令）

仓库根执行：

```bash
just verify-items
```

会用 fresh 默认场景跑完整命令矩阵，打印每条命令的输出与 ✔ / ✖，末尾给场景级摘要。  
实现：[engine/scripts/verify_m1_items.py](../../engine/scripts/verify_m1_items.py)（pytest 也会跑同一套：`tests/test_verify_m1_items_matrix.py`）。

不读、不写 `engine/save`。

## 可选：手动 `just run`

需要自己摸手感或调试单条命令时再用。

1. **先清存档**（否则会恢复旧档，储藏室可能没有夹具）：

```bash
rm -rf engine/save
just run
```

2. 夹具在**储藏室**。进入：

```text
open south
s
```

（方向简写只有 `n/s/e/w`；写全称须用 `go south`。）

## 命令面约定

| 规范动词 | 别名 | 说明 |
|---|---|---|
| `get` | `take` | 对齐 LPC `cmds/std/get.c` |
| `drop` | — | 支持 `drop <物> [数量]`、`drop all` |
| `put` | — | `put <物> in <容器>` |
| `look` / `l` | | 无参看房间；有参看物品详情（含数量/价值/重量） |
| `inventory` / `i` | | 物品栏；堆叠数以 `铜钱×8` 形式展示（数量为 1 时不带 ×） |

标志位（YAML）：**`no_get` / `no_drop`**（对齐 LPC `no_get`，不是 `no_take`）。

## 1. 基线（庭院）

| # | 输入 | 期望要点 |
|---|---|---|
| 1 | `look` | 见「石头」；南出口标注（关） |
| 2 | `i` | 你什么都没带 |
| 3 | `get 石头` | 你拿起 石头 |
| 4 | `i` | 带着：石头 |
| 5 | `drop 石头` | 你放下 石头 |
| 6 | `take 石头` | 与 get 相同（别名） |
| 7 | `drop 石头` | 放下 |
| 8 | `get 石` | 别名「石」可拿 |

## 2. 进入储藏室

| # | 输入 | 期望要点 |
|---|---|---|
| 1 | `open south` | 打开南门 |
| 2 | `s` | 进入储藏室；地面有木箱、铜钱×2 堆、宝石、小布袋、大石块、令牌、石碑 |

## 3. 堆叠 + look 数值

| # | 输入 | 期望要点 |
|---|---|---|
| 1 | `look 铜钱` | long；数量：5（同名两堆去歧义，先看到一堆） |
| 2 | `get 铜钱` | 拿走并合并两堆 |
| 3 | `i` | `你带着：铜钱×8` |
| 4 | `look 铜钱` | 数量：8；价值：1；重量 |
| 5 | `drop 铜钱 2` | 放下 2 个；`i` 见 `铜钱×6` |
| 6 | `get 铜钱 2` | 拿起 2 个；`i` 见 `铜钱×8` |
| 7 | `get 铜钱 99` | 拒绝（没有那么多 / 数量相关提示） |

## 4. 容器 put / get from

| # | 输入 | 期望要点 |
|---|---|---|
| 1 | `get 宝石` | 拿起 |
| 2 | `put 宝石 in 木箱` | 放进木箱 |
| 3 | `look 木箱` | 里面有：宝石 |
| 4 | `get 宝石 from 木箱` | 从箱中取出 |

## 5. 标志位 no_get / no_drop

| # | 输入 | 期望要点 |
|---|---|---|
| 1 | `get 石碑` | 「那东西拿不起来。」（仍在地面） |
| 2 | `get 令牌` | 拿起 |
| 3 | `drop 令牌` | 「这是任务物品，不能丢弃。」 |
| 4 | `put 令牌 in 木箱` | 同上（离开玩家也走 no_drop） |

## 6. 容量 / 重量

| # | 输入 | 期望要点 |
|---|---|---|
| 1 | `get 大石块` | 拿起 |
| 2 | `put 大石块 in 小布袋` | 「太重了，放不进去。」 |
| 3 | `get 宝石`（若已在箱/栏则先放到手）→ `put 宝石 in 小布袋` | 成功（轻且占 1 槽） |
| 4 | （可选）再往小布袋塞第二件 | 容量满则拒绝 |

## 7. get all / drop all

建议清档后重新进入储藏室再测本组。

| # | 输入 | 期望要点 |
|---|---|---|
| 1 | `get all` | 「捡好了。」；石碑仍在地面；其余可拿物进栏；铜钱合并为一堆 |
| 2 | `i` | 有令牌、木箱等；**无**石碑 |
| 3 | `drop all` | 「放下了。」；令牌仍在栏内；其余回地面 |
| 4 | `i` | 仅令牌（或仍持有的 no_drop 物） |

## 8. 门钥匙回归（勿破坏）

从庭院：

| # | 输入 | 期望要点 |
|---|---|---|
| 1 | `n` | 长廊；见铁钥匙 |
| 2 | `get 铁钥匙` | 拿起 |
| 3 | `unlock north` | 解锁 |
| 4 | `open north` | 打开 |
| 5 | `n` | 进入静室 |

## 预期缺失（不当 bug）

- `give <物> to <人>`
- `eat` / `drink` / `wield` / `wear` / `unwield` / `remove`（完整效果层属 M2）
- LPC 语序 `get <数量> <物>`：本引擎为 **`get <物> <数量>`**

## 若矩阵失败

把 `just verify-items` 里 ✖ 的命令、实际输出、期望子串发回来，在本分支修 + 补测。
