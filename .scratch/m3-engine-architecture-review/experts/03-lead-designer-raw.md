## Executive summary

截至 M3，引擎已具备一条**可演示的武侠 MVP 地理旅程**：华山村教程 → 扬州买卖骑马 → 野外遭遇 → 渡口 → 少林拜师练功，并由 `test_m2_e2e_script` / `verify_m2_*` 锁住。机制面上，战斗七步管线、技能学习练习、金钱商店、门派框架、坐骑地形、渡船动态出口、死亡惩罚复活、包外 UGC 加载均已落地。

从主策划视角，当前最大缺口不是「房间不够多」，而是三类会直接伤害**可重复游玩**的断链：

1. **状态 / 效果生命周期（MVP 必做 #31）几乎未做**：ADR-0004 要求的 Effect 调度 / 衰减 / 堆叠仍停留在 tick 预留与「中毒示范文案」；武侠体感中的毒、busy、运功护体、持续 buff 无法成立。
2. **昏迷容错半成品**：规范故事写了「昏迷后自然恢复 / 被治疗」，实现只有「昏迷锁行动 → 再吃一轮伤害才死而复生」；免死区外第一次倒下会把玩家卡死，除非再被打一次。
3. **装备 / 消耗 / 社交表达面空心**：`Equippable`/`Consumable` 占位无 `wear`/`wield`/`use`；频道与表情（MVP 必做 #4/#10）为零；登录（#15）被 REPL 直入替代。单机验证期可接受，但会让「武侠 MUD」读起来像机制沙盘而非江湖。

内容侧：票 10 的房间清单基本齐，但扬州镖局 / 茶馆 / 客栈 / 武庙多为**布景 NPC**；少林达摩院 / 藏经阁无互动深度；整条主线**无任务目标、无疗伤闭环、无坐骑休整**。建议停住 M3 期间优先补「效果骨架 + 昏迷苏醒 + 最小装备 / 消耗 + 场景内容加厚」，**明确不碰**账本 / 创作者平台 / 留言板。

## MVP 场景走查缺口

走查路径对齐票 10 + `m2_mvp_scene.yaml` 连通图 + `TestM2EndToEndScript`：  
**华山村口 → 向导 / 练武场 → 官道 → 扬州 → 马厩 → 东门官道 → 陡坡 / 密林 → 渡口 → 少林山门 → 武场。**

### 1. 华山村（新手）

| 预期（票 10 / M2 US） | 现状 | 缺口 |
|---|---|---|
| 中立出生、学基础命令 | 出生 `huashan_birth`；向导 `ask` 覆盖移动 / 查看 / 拾取 / 战斗 / 去哪 | **无结构化新手进度**（无「已学会」标记、无强制引导顺序）；玩家可跳过直接出村 |
| 低风险战斗教学 | 练武场 `NoDeathZone` + 稻草人（不反杀） | 木桩可交战；**打到昏迷后在免死区会反复昏迷且无法行动**，无苏醒 / 疗伤出口，教学区可能变成软卡死 |
| 拾取练习 | 小石子 | 成立；价值弱，无后续用途 |

### 2. 扬州（城镇丰富子集）

| 地标 | 现状 | 玩法缺口 |
|---|---|---|
| 广场 + 四街四门 | 房间齐全，西门双官兵可序号消歧 | 城门官兵无盘查 / 阻挡玩法，纯布景 |
| 钱庄 | 伙计卖银票 | **不是存取款**；「钱庄」名实不符，削弱城镇金融体感 |
| 打铁铺 | 可买钢刀 | 刀进背包即被门禁视为「持刃」（无 wield 语义）→ **买了武器反而进不了少林**，体验荒诞 |
| 客栈 / 茶馆 / 镖局 / 武庙 | 有 NPC + 默认 inquiry | **零交易 / 零任务 / 零休息恢复**；票 10「商业氛围」只剩文案 |
| 马厩 | 50 两买黄骠马，可骑 | 成立；城里骑乘规则未强调（可室内骑？原版启发是户外限制，当前靠内容未卡） |

### 3. 野外 + 官道 + 渡口 + 坐骑

| 环节 | 现状 | 缺口 |
|---|---|---|
| 华山↔扬州官道 | `road_huashan_yz` 连通 | 成立；无遇敌 / 无事件 |
| 东门外矮种马 | ability=2，陡坡 cost=8 骑不过 | **地形教学成立** |
| 密林山贼 | aggro + loot + respawn | 成立；**无败北后的可读叙事**（昏迷卡死风险同上）；单怪密度固定 |
| 渡口 | 周期动态 `across` + look 提示 | 成立；等船是唯一「时间玩法」亮点 |
| 坐骑精力 | 骑乘扣 jingli，归零摔下昏迷 | **无喂养 / 草地恢复 / 马厩休整**；长途第二次骑乘易废 |

### 4. 少林寺（门派）

| 环节 | 现状 | 缺口 |
|---|---|---|
| 山门 EntryGuard | 男 / 无他派 / 非持刃 | 机制成立；持刃判定=背包含 edged 标签，**过严且不可「收刀」**（无 unwield） |
| 拜师 join | 知客僧提示 + `join 少林` | 成立；无拜师仪式 / 辈分 |
| 武场 learn / practice | 罗汉拳 + 混元一气功（map_skill） | 成立；混元招式 damage_type=none，**内功几乎无战斗存在感** |
| 达摩院 / 藏经阁 | 展示 NPC | **剧情纵深=只读文案**；无阅经 / 面壁 / 试炼 |

### 5. 闭环总评

- **地理闭环**：已通，可验收「一张连通的 MVP 地图」。  
- **动机闭环**：弱——没有任务、成就、门派贡献、经济目标以外的「下一步」。  
- **风险闭环**：半通——能死能复活，但昏迷中间态缺出口。  
- **成长闭环**：浅通——能练级打架，缺装备成长与状态博弈。  
- **UGC 闭环（M3）**：与武侠 MVP 正交已打通；示例包故意不用战斗，**不证明**武侠内容包可被外部作者扩展出同等深度。

## 联动玩法风险点

| ID | 联动链 | 当前覆盖 | 风险 |
|---|---|---|---|
| L1 | 战斗 → 技能招式 → **效果持续** → 再战斗 | 战斗↔技能数值通；效果断 | 无法做「中毒压血 / 封招 / 护体」；武侠差异化全靠 force 数字 |
| L2 | 装备 → 属性 / 伤害 → 战斗；装备 → 门禁 | 门禁用 `ItemTags`；装备不进 PowerModel | 钢刀只影响进寺，不影响打架；买装备动机扭曲 |
| L3 | 门派 → map_skill → learn/practice → 战斗选招 | 通 | 浅但稳；内功技能未参与 AP/DP 是内容洞，非机制洞 |
| L4 | 金钱 → 商店 → 坐骑 / 武器 → 旅行 / 门禁 | 通但副作用坏 | 买刀挡少林；花钱无成长感 |
| L5 | 战斗 → 昏迷 / 死亡 → 复活 → 再探索 | 死→复活通；**昏迷无出口** | 野外第一次倒下即软锁（除非再挨打触发死亡） |
| L6 | 坐骑 jingli → 地形 cost → 摔下 Unconscious | 通；坐骑昏迷后无恢复链 | 长线旅程第二次骑乘失败率高 |
| L7 | 渡船 tick → 动态出口 → 跨区域 | 通 | 低风险；是当前最好的「时间×空间」联动示范 |
| L8 | aggro NPC → 交战 → loot → respawn | 通 | 低；无「清场后安全窗」设计问题 |
| L9 | Nature 时辰 → 条件 / AI / 商店 | Nature 有；内容未消费 | 条件语言能力闲置 |
| L10 | 任务 / 对话旗标 → 房间解锁 / 奖励 | **无任务状态机** | 无法把扬州空壳地标变成可玩目标，只能继续堆 inquiry 文案 |
| L11 | UGC 包 → 武侠同构能力 | M3 示例避开战斗 / 技能 / 坐骑 | 外部作者复制 MVP 武侠深度时，会撞上 L1/L2/L5 同一批洞 |

## 改进建议 P0 / P1 / P2

> 范围边界：**停住 M3 期间**可做的内容 / 系统补强。  
> **明确不包含**：M4 账本 / 分成 / 埋点、创作者 Web 平台、留言板、完整阴间、PvP、武林大会、21 门派、商业化双货币。

### P0 — 不补则「可玩」含金量不稳（建议本评审后优先排期）

1. **昏迷苏醒最小闭环**  
   - 免：免死区或野外第一次倒下后无法行动、无自然恢复。  
   - 建议：tick 倒数苏醒（或房间休息命令）+ 苏醒时恢复少量气血；可选极简 `heal`/`rest`。  
   - 验收：练武场打晕能醒；野外晕倒若不被补刀可自行醒来。

2. **Effect 生命周期骨架（兑现 ADR-0004 / 必做 #31 的最小切片）**  
   - 交付：挂实体的效果表 + tick/wallclock 衰减 + stacking 四枚举之一二种 + 至少一个真实持续效果（如每跳掉血的毒）。  
   - SkillBehavior 应能「上效果」而不只改瞬时伤害。  
   - 验收：示范招式上毒 → 数 tick 掉血 → 到期清除；存档进出仍正确（或明确瞬时不进档）。

3. **持刃门禁与装备语义对齐**  
   - 短期内容修补：少林门禁改为「正在握持」或「未放入容器」；或提供 `stash`/`pack` 收刀。  
   - 或 P0.5 上最小 `wield`/`unwield`（只服务门禁 + look，不要求九槏）。  
   - 验收：买钢刀仍可进少林（收刀后）；持刃时拒绝文案可读。

### P1 — 明显提升武侠 MVP 体感（内容为主、系统为辅）

4. **扬州空壳地标加厚（纯内容优先）**  
   - 客栈：`rest` 恢复气血 / 内力（可绑 P0 苏醒）。  
   - 钱庄：存取款命令或至少「兑银票↔银两」双向，名实相符。  
   - 镖局：一条最小「接镖→指定房间交货→领赏」脚本任务（可先硬编码旗标，不建通用任务引擎）。  
   - 茶馆 / 武庙：2～3 条有奖励的情报 inquiry（银两 / 经验 / 去少林提示）。

5. **消耗品最小可用**  
   - `use`/`eat` 吃金创药回血；Consumable.uses 真正扣减。  
   - 商店上架；与 P0 昏迷形成「用药醒来」。

6. **坐骑休整**  
   - 马厩 `rest`/`feed` 回 jingli；或草地房间自动回复（内容配置）。  
   - 验收：摔马后回扬州能再次骑乘走完东线。

7. **表情 / 消息模板最小集（必做 #10 的瘦身版）**  
   - 不建完整词库也可：战斗 / say / look 共用 `$N`/`$n` 替换，为多人预留。  
   - 频道（#4）若坚持单机，应在评审结论里**正式改判为「多人里程碑必做」**，避免清单与现实长期分裂。

### P2 — 增强深度，可排在 P0/P1 之后

8. 内功参与 PowerModel（或 practice 外的 `exert` 短时 buff）。  
9. 少林剧情房：藏经阁读经得临时效果；达摩院一次试炼战斗。  
10. NPC 战斗喊话 / 残血逃跑（AI 现代化改造的小切片）。  
11. `give` + 简易交易；同名消歧扩到 get/look。  
12. 新手引导：向导发放「出门包」（药 + 一点银）+ 完成三连问答给奖励。  
13. 登录 / 创角：仅当准备联网试玩时再开；停 M3 评审期可不做。

## 待交叉对抗争议点（≥5）

供架构 / 战斗系统 / 内容 / 工程专家对撞；主策立场已括号注明。

1. **状态系统是否仍算「M2 已交付的 MVP 必做」？**  
   （主策：否——钩子示范 ≠ Effect 生命周期；应记为债务或正式缩小 ADR-0004 MVP 切片。）

2. **频道 / 表情在单机阶段是否应从「MVP 必做」改判？**  
   （主策：建议改判为「多人上线门禁」；否则清单长期造假。若坚持必做，则 P1 必须排期。）

3. **昏迷无苏醒是 bug 还是可接受的「两段式简化」？**  
   （主策：对玩家是 bug 级体验；对测试用「再敲一下强制死」掩盖了产品问题。）

4. **持刃门禁用背包标签是否可接受？**  
   （主策：不可接受为长期设计；要么上 wield，要么改条件语义。）

5. **扬州「丰富子集」是否已达成票 10 意图，还是仅「房间清单打勾」？**  
   （主策：地理打勾、玩法未达标；应用「每地标至少一条可验证互动」做补验收。）

6. **停住 M3 是否应插入「M2.5 可玩性加固」再进 M4？**  
   （主策：是；M4 商业数据模型不解决 L1/L5，先补玩法再堆商业更稳。）

7. **任务要不要上通用引擎，还是内容硬编码三条剧本？**  
   （主策：停 M3 期用硬编码旗标即可；通用任务引擎易膨胀，留给后续里程碑。）

8. **M3 非武侠示例包是否掩盖了武侠题材包的表达力缺口？**  
   （主策：有掩盖风险；应另备一份「外部武侠微包」验证战斗 / 门派 / 坐骑字段，而不是只验门锁商店。）
