## 共识点（四人高度一致）

下列结论在至少三位专家正文中独立出现，且无一方提出实质反驳；对抗阶段予以**确认保留**。

### C1. M3「包外声明式 → 加载 → 校验 → 可玩一次」已按刻意收窄的承诺兑现

- **架构师** Executive summary / 优点第 3 条：M3 组合 `load_scene`、零改字段集、非武侠包证明题材无关加载。
- **UGC** 符合度矩阵 A1–D4：**满分档**相对 M3 刻意收窄承诺；Out of Scope 无蔓延。
- **主策划** 闭环总评：UGC 闭环与武侠 MVP 正交已打通。
- **规格 QA** 里程碑矩阵 M3：**符合**；`test_m3_pack_loop` / `just verify-m3` 锁死。

**对抗结论**：可宣布 **M3 里程碑交付**（相对其自身 spec），不因「不是完整 UGC 产品」而否定 M3。

### C2. ADR-0004「Effect 生命周期」存在文档/实现落差，禁止静默悬空

- **架构师** R1：全库无 `Effect`/`StackingPolicy` 实现；M2-16 允许文案顶替。
- **主策划** 系统对照 #31：**未实现（严重缺口）**；L1 联动断链。
- **规格 QA** 漂移 D1：**高**；符合度 M2 标「部分→接近符合」。
- **UGC** 未以 Effect 为主线，但承认声明式天花板与脚本后置——与「状态机制缺席」相容。

**对抗结论**：四方共识「必须显式决策」。见分歧矩阵 D1 与下文裁决（实现最小骨架 **或** 正式改判 ADR，二选一）。

### C3. 停住 M3：明确不做商业化实现与创作者平台 / 编辑器

- 四方 Executive / P2 / Out of Scope 均切割 [ADR-0006](../../../docs/adr/0006-no-engine-editor-board-post-mvp-creator-platform.md)、[post-mvp-backlog](../../mvp-scope/post-mvp-backlog.md)、用户「先停 M3」。
- **UGC** / **架构师**：`PackManifest` 元数据雏形已够「留位置」；勿空实现账本/埋点。

**对抗结论**：停机窗口 **禁止** 开 M4 账本、Web 平台、留言板、RestrictedPython。

### C4. ADR-0001 边界守住：侠客行对照 = 灵感，≠ 规格债

- 四方均设专节标注「非规格」；**规格 QA** 测试原则锚定自身契约。
- **主策划** / **架构师** 对照表明确「不像武侠」≠「必须补齐才算完成侠客行」。

**对抗结论**：综合报告缺口节必须继续双栏标注；不得把频道/阴间/21 门派等灵感项升格为停机阻塞。

### C5. 测试分层与门禁健康（649 绿 + verify 双轨）是停机可信度的工程底座

- **规格 QA** 覆盖地图与 verify↔pytest 回挂为强项。
- **架构师** / **主策划** / **UGC** 均引用 649 绿与 e2e/verify 作为「主路径可玩」证据。

**对抗结论**：停机不因「缺测试资产」而失败；风险在契约诚实度与少数联动断链，不在覆盖真空。

### C6. 昏迷无自然苏醒（相对 M2 US23）是产品/规格双面问题

- **主策划** L5 / P0.1：软卡死；免死区教学可能卡死。
- **规格 QA** D2 / 联动表：US23 **未达标**。
- **架构师** 未单列但死亡流程描述与「仅二次打击」一致。

**对抗结论**：属 P0——要么补苏醒，要么改用户故事锁定「无自然苏醒」语义；不可继续用「再挨打」掩盖。

### C7. 引擎不做编辑器；创作面 = 手写 YAML + CLI（诚实边界）

- **UGC** / **架构师** / **主策划** / **规格 QA**（K4）一致对齐 ADR-0005/0006。

| 「extension_data 供规则引擎」 | 代码注释历史表述 | M3 明确不消费 | 标 `RESERVED_UNUSED`；伪扩展点降级 |

## 对抗阶段结论（给 final 报告的输入）

1. **可以宣布 M3 里程碑交付**，但必须附带：**规格诚实加固未完成前，不得宣称 ADR-0004 字面与「18 个 MVP 必做」已全部兑现**。
2. **可以停住、暂不进入 M4 / 创作者平台**；停机窗口应消化上表 P0（决策类可当天落文档，实现类可开加固票）。
3. **最大真分歧**已收敛为可执行选择：Effect 与昏迷苏醒各自「实现或改规格」；其余多为优先级与叙事问题。
4. **灵感缺口**（频道、完整武学博弈、阴间等）不得阻塞停机，但须与规格债分栏。
