## 一、总览

### 目标

按 [08 阶段 0 任务 1](08-阶段-0-实施计划.md) 提取的 9 层 LPC 规格契约，逐个子系统实现 runtime，每个子系统通过行为等价验证（hypothesis 属性测试 + golden trace diff）。阶段 2 结束时，核心可玩循环（拜师/练功/战斗/任务/死亡轮回）在 greenfield 引擎上可跑，核心引擎无武侠烙印（[04 §五检查点 8](04-迁移路径与避坑清单.md)）。

### 2.1：Query/索引层 + Identity/Position/Inventory（M2-1，低风险）

- **目标**：实现 LPC `query()`/`query_temp()`/`set()`/`set_temp()` 运行时语义，对照 [13-dbase-key-map.md](13-dbase-key-map.md) 映射表。完善 Identity/Position/Inventory 组件规格等价。
- **输入**：T3 [DBASE_KEY_MAP](../../engine/src/xkx/runtime/dbase_map.py)（37 已映射 key + 2 路径前缀 + 55 后置）+ [层 B 对象基础](08-阶段-0-实施计划.md)规格（24 函数）
- **产出**：
  - 运行时 `query(eid, key)` / `query_temp(eid, path)` / `set(eid, key, val)` 接口（对照 LPC F_DBASE）
  - 索引层：`entities_in_room` 已有，扩展 `entities_with_family` / `entities_by_prototype` 等
  - Identity/Position/Inventory 组件规格等价（short()/present()/move() 语义）
  - 55 后置 key 逐步激活（按"实现到时才补"原则）
- **依赖**：阶段 1 T3 DBASE_KEY_MAP ✅
- **验收**：LPC `query("skill/axe")` -> `Skills.levels["axe"]` 语义等价；hypothesis 属性测试（路径前缀解析 + 未映射 key 返回 None）
- **需 ADR**：ADR-0025 Query/索引层设计（query() 语义 + 索引 + 后置 key 激活策略）

### 2.2：Vitals/Heal/Condition（M2-2，低风险）

- **目标**：实现 LPC `heal_up`（自然恢复）+ 扩展具体 condition 类型（蛇毒/醉/失明），对照 [层 F 死亡轮回](08-阶段-0-实施计划.md)规格（10 函数）+ condition 框架。
- **输入**：T1 [ConditionSystem](../../engine/src/xkx/runtime/conditions.py)（框架已落地）+ 层 F 规格（die vs unconscious + death_penalty + make_corpse）+ [层 E](08-阶段-0-实施计划.md) 三层资源不变量
- **产出**：
  - HealSystem（tick 驱动自然恢复，LPC `heal_up` 每 2s + food/drink 修正）
  - 具体 condition 类型（蛇毒 DoT / 醉 debuff / 失明）填充 EffectComp
  - 死亡轮回（die -> make_corpse 物品转移 + death_penalty 经验扣除 + unconcious 昏迷升级）
  - 三层资源不变量验证（0<=qi<=eff_qi<=max_qi）
- **依赖**：2.1 Query（Vitals 组件 query）+ T1 ConditionHandler 契约 ✅
- **验收**：heal_up 数值等价 LPC；condition DoT 周期触发 + 衰减；die vs unconcious 触发条件正确
- **需 ADR**：无（T1 ADR-0018 契约已定，2.2 是填充实现）

### 2.3：Attribute/Skill/Equipment（M2-3，中风险）

- **目标**：实现 ModifierStack 三类语义 + 技能三层划分 + 装备槽，对照 [层 E](08-阶段-0-实施计划.md) skill_power 公式 + LPC equip/unequip。
- **输入**：阶段 1 [Skills](../../engine/src/xkx/runtime/components.py) 组件（levels + apply_*）+ 层 E skill_power 公式（level³/3 + jingli_bonus + str/dex 加成）
- **产出**：
  - ModifierStack（临时修正 apply_* / 装备加成 / 永久基础值，三类叠加语义）
  - 技能三层划分（基础技能 unarmed/dodge/parry / 武学技能 kungfu/*/  特殊技能）
  - Equipment 组件（装备槽 weapon/armor/accessory + equip/unequip + 负重）
  - skill_power 完整公式接入 ModifierStack
- **依赖**：2.1 Query（Attribute/Skills 组件 query）
- **验收**：ModifierStack 三类叠加与 LPC `query_temp("apply/attack")` + 装备加成等价；技能三层划分清晰
- **需 ADR**：ADR-0026 ModifierStack 三类语义 + 技能三层划分

### 2.4：Combat（M2-4，高风险★）

- **目标**：补全阶段 1 T6 剩余项 + 行为等价验证 + 文本体验流 diff。
- **输入**：阶段 1 [resolve_attack.py](../../engine/src/xkx/combat/resolve_attack.py)（七步管线已落地）+ [层 E](08-阶段-0-实施计划.md) 26 函数规格 + golden trace（driver 可运行）
- **产出**：
  - 闭包型 call_out（144 处）翻译为 EffectComp（可中断/可崩溃恢复）
  - s_combatd.c 阵法合击规格 -> CombatModifier（多人协同攻击修正）
  - hit_ob/hit_by 回调 + 技能 action 武侠武学数据填充（SkillData 载体已有）
  - 数值回归基线（golden trace 对比 do_attack 七步时序 + 31 处 random 概率分布）
  - 文本体验流 diff（同 seed 同 input -> 文本输出与 LPC golden trace diff）
- **依赖**：2.3 Attribute/Skill/Equipment（skill_power 接入 ModifierStack）+ 阶段 1 T6 ✅
- **验收**：ConformanceChecker 8 项全通过 + golden trace diff 无语义差异 + 文本体验流 diff 可接受
- **需 ADR**：ADR-0027 call_out -> Effect 翻译 + 阵法合击 CombatModifier
- **kill criteria**：行为等价验证失败 -> 先优化 combat-sim（补全简化台账）-> 仍失败触发 kill criteria（暂停 Combat 迁移，回退阶段 1 T6 状态）

### 2.5：TitleSystem 称谓（M2-5，中风险）

- **需 ADR**：ADR-0028 RANK_D 规格提取 + PronounContext 三元组完整求值

### 2.6：WorldGovernanceSystem（M2-6，中风险）

- **目标**：实现 1-2 个代表性治理元素（阴间 / 法院），对照 [09 灵魂系统盘点](09-灵魂系统盘点.md)。平台级 fail-closed Python，不落入 UGC 可编辑规则层。
