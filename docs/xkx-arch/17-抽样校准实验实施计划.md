# 抽样校准实验实施计划（阶段 0 任务 6）

> 阶段 0 任务 6（抽样校准实验）的实施计划。任务 6 是 [04](04-迁移路径与避坑清单.md) 定义的阶段 0 可后置可选项，M3 收官后作为可穿插推进项启动。
> 方法论见 [ADR-0046](../adr/ADR-0046-sampling-calibration-methodology.md)。

## 一、目标

68771 call_other 调用点（[00](00-愿景约束与总纲.md) §11）按难度分层抽 50-100 个实测工时，为全量迁移工时承诺提供数据支撑（[04](04-迁移路径与避坑清单.md) 验收"工时承诺有数据支撑"）。分 A/B 两阶段（[ADR-0046](../adr/ADR-0046-sampling-calibration-methodology.md)）：

- **阶段 A**（本轮）：枚举 + 分布统计 + 抽样方案 + 迁移单位建议
- **阶段 B**（后续）：手工实测样本工时 + 全量推算

## 二、阶段 A 步骤

### 1. 枚举脚本 `engine/tools/sampling/scan_callothers.py`

遍历全仓库 `.c` 文件（8151），扫描 `->` call_other：

- 形式：`obj->method(args)` / `obj->"method"(args)`（动态方法名）
- 过滤：字符串字面量（`"..."` / `'...'`）、注释（`//` / `/* */`）
- 提取：文件路径、行号、被调方法名、调用表达式、子系统（顶层目录）
- 输出：JSONL（每行一个调用点）+ 汇总 JSON

### 2. 分类统计

- 子系统分布（d / cmds / adm / kungfu / feature / inherit / clone / 其他）
- 方法名 top-N（query / set / short / long / move / ...）
- 文件聚合（每文件调用点数分布）
- 调用形式（静态方法名 vs 动态字符串）

### 3. 抽样方案

- 分层配额：子系统 × 方法名类别
- 50-100 样本总量
- 每层样本数

### 4. 产出报告 `engine/tools/sampling/report.md`

- 总量验证（对账 68771，口径差异说明）
- 分布统计表
- 抽样方案
- 迁移单位建议（基于分布数据，供阶段 B 拍板）

## 三、风险

| 风险 | 应对 |
|---|---|
| `->` 扫描口径与 [_archive/01](../xkx-arch/_archive/01-关键修正与避坑清单.md) 实测 68771 不一致（注释/字符串过滤规则差异） | 报告对账说明口径差异；以本脚本口径为准（可复现） |
| 动态方法名（`->"func"`）无法静态提取方法名 | 单独统计占比，归入"动态"类别 |
| LPC 语法边角（`@"..."` raw string / `##` 宏）干扰 | 状态机覆盖常见形式；边角 case 在报告遗留 |

## 四、不做（阶段 A 边界）

- 不实测工时（阶段 B）
- 不实现调用点迁移（阶段 B）
- 不扫描 `call_other()` efun 形式 / `.h` 文件

## 六、阶段 B 实施子计划（修正后，[ADR-0048](../adr/ADR-0048-stage-b-degraded-interval-pilot.md)）

> 阶段 B 原 [ADR-0047](../adr/ADR-0047-greenfield-effort-semantics.md) 方案（80 样本窄 t 分布置信区间）经调研 + 架构专家评审团修正为"降级区间承诺 + pilot 纠偏"。本节是修正后的执行计划。

### 1. 方法论修正要点

- 目标降级：80 样本窄 CI -> pilot 12-16 + 类比基准**区间承诺**（04 验收"有数据支撑"非"窄点估 + CI"）
- 分类纠偏：每样本记 `corrected_status`/`corrected_kind`，误分类率定量回调 [ADR-0047](../adr/ADR-0047-greenfield-effort-semantics.md) 阈值
- LLM 辅助否决纳入主线（工时语义污染）；代码落一次性目录
- 详见 [ADR-0048](../adr/ADR-0048-stage-b-degraded-interval-pilot.md)

### 2. pilot 样本选择（12-16 个，从 [sample_candidates.json](../../engine/tools/sampling/output/sample_candidates.json) 80 候选）

high-tier 过采样（工时变异源主体）+ 纠偏样本（校准分类偏差）+ 子系统覆盖：

| 层 | 数量 | 代表样本（含纠偏） |
|---|---|---|
| pending/logic high | 6-8 | `xue.c:main`(cc53) / `do_check_menpai_job`(cc45) / `do_check`(cc42) / `do_qingjiao`(cc40) / `do_copy`(cc36) / `next_sword`(cc34) / `move_liquid`(cc30) |
| pending/logic mid | 2-3 | `do_emote`(cc30) / `do_setorg_default`(cc27) / `bash_weapon`(cc27) |
| pending/data（含纠偏） | 2 | `snake.c:init`（误分类：实为 logic）/ `char.c:setup`（架构替代低工时确认） |
| implemented/logic（含纠偏） | 1-2 | `death_penalty`（后置分支偏差）/ `bai.c:main`（确认≈0 偏差） |

子系统覆盖：d / kungfu / cmds / adm / clone / feature / inherit 各至少 1 个。

### 3. 共享桩预建（pilot 前一次性补建）

评审团发现的缺口（多 logic 样本共用），落 [engine/tools/sampling/pilot/stubs.py](../../engine/tools/sampling/)：

- `SKILL_D->valid_learn` / `type` / `query_skill_name`（SkillData 扩展或桩）
- `recognize_apprentice` / `prevent_learn` / `is_spouse_of` / `receive_damage`

### 4. 工时记录 schema（每样本一条 JSONL）

```json
{
  "file": "...", "func": "...", "subsystem": "...", "status": "...", "func_kind": "...", "tier": "...", "call_count": N,
  "corrected_status": "...", "corrected_kind": "...",
  "misclassified": false, "misclass_reason": "",
  "effort_min": { "read_spec": N, "write_code": N, "write_test": N, "debug": N, "subtotal": N },
  "reusable_api": [], "missing_api": [], "notes": ""
}
```

落 `engine/tools/sampling/pilot/effort_records.jsonl`。

### 5. 推算方法（区间承诺，非窄 CI）

1. 分层（`corrected_status` × `corrected_kind` × `tier`）样本均值 × 层规模 -> 分层点估
2. 类比基准校准：30 文件校准工时 + M3 C1-C6 工时 + 调研 3 个 high-tier 数据点（120-190 min）
3. 输出全量迁移工时区间（点估 + 上下界），非窄 t 分布置信区间
4. 误分类率 -> 定量回调 [ADR-0047](../adr/ADR-0047-greenfield-effort-semantics.md) 分类阈值

### 6. session 分批（总 14-20h / 2-3 session）

- session 1：建共享桩 + pilot SOP/模板 + 测 1-2 个 high-tier（如 `xue.c:main`）
- session 2-3：测剩余 pilot 样本（每 session 1-3 个）+ 推算 + 区间承诺报告

### 7. 退路 / 升级触发条件

- **退路**（升级接力补测）：误分类率 >30% 或 high-tier 工时变异系数 >1.0 -> 回 [ADR-0047](../adr/ADR-0047-greenfield-effort-semantics.md) 分批实测路径
- **升级**（窄 CI 回归外推）：未来对外硬交付日期需窄置信区间 -> pilot 16 样本作回归起点补样到 40-55（[ADR-0048](../adr/ADR-0048-stage-b-degraded-interval-pilot.md) 决策 6，按需触发）
