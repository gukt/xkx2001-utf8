# 侠客行 MUD 现代化重构架构方案

> 本方案由研究 Agents Team（36 个 Agent，四阶段：现状分析 / 模式研究 / 方案设计 / 对抗验证）产出，并经对抗验证修正承重级假设后整合；复经 8 位资深专家（架构师/主策划/制作人/MUD历史/UGC/分布式/引擎/AI）对抗复审与交叉质证修正。

## 重构目标

将经典 LPC MUD《侠客行》（8412 LPC 文件）从 0 到 1 重构为全新项目（LPC 作为规格源，新引擎作为目标）：

- **后端**：Python + WebSocket（asyncio + uvloop）
- **前端**：现代 Web（React + TypeScript）
- **部署**：分布式大规模，承载高并发
- **平台**：UGC 平台，DSL 配置剧情场景
- **创作**：Agent 协作创作多题材世界（武侠 / 大航海时代 / 书院 / 穿越 / 现代剧情）
- **标杆**：业界标杆，可影响整个游戏圈

## 核心立场

1. **垂直切片优先**：最高产品风险（DSL+Agent 创作闭环）先验证，再投入引擎重构。
2. **greenfield 框架**：LPC 是规格源不是运行系统，清除所有"适配层/双栈/过渡期"增量假设。
3. **诚实面对迁移面**：68771 处同步调用是年级工作量，需抽样校准。
4. **规格提取先行**：从 LPC 提取形式化规格，按规格从零实现，行为等价验证。
5. **性能以实测定**：阶段 0 做 μs 级基准 + tick 预算，按结果决定是否引入 compiled core。
6. **标杆靠差异化**：DSL-as-contract Agent 闭环 + 全仿真确定性 + themed 治理 + 开源生态，而非"把 LPC 迁到 Python"。
