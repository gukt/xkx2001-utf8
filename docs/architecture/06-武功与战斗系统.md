## 1. 武功体系概览

武功系统是《侠客行》的核心玩法，整个系统代码分布在 `/kungfu/` 目录下，包含三个主要子目录：`class/`（门派）、`skill/`（技能）和 `condition/`（状态）。

### 1.1 门派系统 (class/)

《侠客行》包含21个门派，每个门派有独立的子目录，存储该门派的NPC定义和技能配置。

#### 典型门派分析

以**少林派** (kungfu/class/shaolin/)为例：

- **NPC组织**：包含30+个NPC文件，按辈分划分
  - "玄"字辈：玄苦（罗汉堂首座
  - "道"字辈：道字辈高僧
  - "慧"字辈、"澄"字辈弟子
  - "清"字辈弟子
  
- **门派特色技能**：
  - 拳法：罗汉拳、金刚拳
  - 掌法：般若掌、三花聚顶掌
  - 指法：拈花指、一指禅
  - 内功：混元一气功
  - 轻功：少林身法
  
- **门派特殊机制**：
  ```c
  // 少林派拜师条件
  // 仅限僧人，有严格的辈分要求
  // 可通过闯十八罗汉大阵获得越级拜师资格
  ```

以**武当派** (kungfu/class/wudang/)为例：

- **NPC组织**：
  - 知客道长：负责门派接待和守卫山门任务
  - 各种道长、道童
  - 有完整的师门传承体系

- **门派特色技能**：
  - 剑法：太极剑
  - 掌法：绵掌
  - 内功：太极神功
  - 轻功：梯云纵

- **门派任务系统**：
  - 守卫山门任务
  - 敬香系统
  - 香火贡献度机制

#### 门派与技能的绑定方式

1. **技能映射 (map_skill)**：
   ```c
   // 将基础技能类型映射到门派特定技能
   map_skill("force", "hunyuan-yiqi");    // 内功->混元一气功
   map_skill("dodge", "shaolin-shenfa");  // 轻功->少林身法
   map_skill("cuff", "luohan-quan");     // 拳法->罗汉拳
   ```

2. **技能准备 (prepare_skill)**：
   ```c
   // 准备战斗中使用的拳脚技能
   prepare_skill("cuff", "luohan-quan");
   ```

3. **技能限制**：
   - 特定技能仅限特定门派学习
   - 高级技能有基础技能等级要求
   - 某些技能有特殊限制（如性别、是否出家）

### 1.2 技能系统 (skill/)

技能系统包含97个技能目录，每个技能都是一个独立的LPC对象。

#### 技能分类

1. **武器技能**：
   - sword（剑法）
   - blade（刀法）
   - club（棍法）
   - axe（斧法）
   - whip（鞭法）
   - finger（指法）
   
2. **拳脚技能**：
   - strike（掌法）
   - cuff（拳法）
   - claw（爪法）
   - unarmed（空手）

3. **防御技能**：
   - parry（招架）
   - dodge（躲闪）

4. **内功技能**：
   - force（内功基础）
   - 各种门派特有内功

5. **其他技能**：
   - literate（读书识字）
   - taoism（道法）
   - buddhism（佛法）
   - begging（乞讨）

#### 技能数据结构

以般若掌 (banruo-zhang.c)为例，技能包含以下核心要素：

```c
mapping *action = ({
    ([
        "action" : "招式描述",      // 攻击动作描述
        "force" : 150,                // 内力消耗/威力
        "dodge" : 5,                  // 躲闪难度修正
        "damage_type" : "瘀伤",       // 伤害类型
        "lvl" : 0,                   // 需求等级
        "skill_name" : "横空出世"      // 招式名
    ]),
    // 多招按等级排列
});
```

以六脉神剑 (6mai-shenjian.c)为例，高级技能具有特殊结构：

```c
mapping *action = ({
    ([
        "action" : "左手小指点出，使一招「少泽剑」...",
        "force" : 300,
        "dodge" : 15,
        "damage": 350,              // 额外伤害
        "lvl" : 22,
        "skill_name" : "少泽剑",
        "weapon": "无形剑气",       // 虚拟武器
        "damage_type" : "刺伤"
    ]),
});
```

#### 技能核心接口

每个技能继承 `/inherit/skill/skill.c`，提供以下标准接口：

| 函数 | 用途 |
|-----|-----|
| `valid_enable(string usage)` | 检查该技能能否用于某种用途 |
| `valid_learn(object me)` | 检查角色能否学习该技能 |
| `query_action(object me, object weapon)` | 获取当前可用招式 |
| `skill_improved(object me)` | 技能升级时触发的效果 |
| `perform_action_file(string action)` | 返回绝招文件路径 |
| `exert_function_file(string func)` | 返回内功功能文件路径 |
| `hit_ob(object me, object victim, int damage_bonus)` | 击中时的特殊效果 |

#### 技能等级与经验系统

技能学习与提升流程：

1. **学习条件**：
   - 需要基础属性达到要求
   - 有时需要前置技能达到一定等级
   - 部分技能有内力/性别等特殊限制

2. **练习机制**：
   ```c
   // practice_skill()提供练习消耗
   me->receive_damage("jingli", 消耗);
   me->add("neili", -消耗);
   ```

3. **技能提升奖励**：
   - 某些技能在特定等级会给属性加成
   - 高级技能可能解锁特殊功能

### 1.3 技能基类 (inherit/skill/skill.c)

`skill.c` 是所有技能的基类，定义了技能系统的核心框架。

#### 主要功能

2. **技能类型**：
   ```c
   string type() { return "martial"; }
   // 有 "martial"（武术）和 "knowledge"（知识）两种
   ```

3. **技能提升回调**：
   ```c
   void skill_improved(object me) {}
   // 技能升级时自动调用
   ```

4. **绝招系统**：
   ```c
   int perform_action(object me, string arg) {
       // 解析绝招命令，调用相应文件
   }
   ```

5. **内功功能系统**：
   ```c
   int exert_function(object me, string arg) {
       // 解析内功命令，调用相应文件
   }
   ```

6. **武器淬毒**：
   ```c
   mixed hit_ob(object me, object victim, int damage_bonus, int factor) {
       // 击中时自动施毒
   }
   ```

#### 技能随机系统

基类提供了独特的随机数生成器，用于选择招式：

    // 生成加权随机数，高级招式有更大权重

## 2. 战斗系统核心

战斗系统由三层组成：
1. `feature/attack.c` - 角色战斗状态管理
2. `feature/damage.c` - 伤害与生命系统
3. `adm/daemons/combatd.c` - 战斗仲裁

### 2.1 战斗状态管理 (feature/attack.c)

#### 核心数据结构

```c
static object *enemy = ({});    // 敌人列表
static string *killer = ({});   // 必杀对象id
```

#### 战斗状态函数

| 函数 | 功能 |
|-----|-----|
| `is_fighting(object ob)` | 检查是否在战斗 |
| `is_killing(string id)` | 检查是否必杀某人 |
| `fight_ob(object ob)` | 开始战斗 |
| `kill_ob(object ob)` | 开始死战 |
| `remove_enemy(object ob)` | 停止战斗 |
| `remove_killer(object ob)` | 取消必杀 |
| `remove_all_enemy()` | 清除所有敌人 |
| `remove_all_killer()` | 清除所有必杀 |

#### 对手选择机制

```c
object select_opponent() {
    // 最多同时与4个敌人战斗
    // 随机选择一个对手
    int which = random(MAX_OPPONENT);
    return which < sizeof(enemy) ? enemy[which] : enemy[0];
}
```

#### 战斗动作重置

```c
void reset_action() {
    // 根据武器/准备的技能更新战斗动作
    // 调用技能的query_action()获取招式描述
}
```

#### 心跳攻击流程

```c
int attack() {
    object opponent = select_opponent();
    
    // 清理无效敌人
    clean_up_enemy();
    
    // 选择对手并攻击
    if (objectp(opponent)) {
        set_temp("last_opponent", opponent);
        
        // 检查是否投降
        if (this_object()->query_temp("yield")) return 1;
        
        // 首先尝试特殊战斗（如双手互博）
        if (!special_attack(opponent)) {
            // 调用COMBAT_D进行标准战斗
            COMBAT_D->fight(this_object(), opponent);
        }
        return 1;
    }
    return 0;
}
```

#### 自动战斗触发

```c
void init() {
    // 当有其他对象进入时检查是否需要自动战斗
    // 1. 是必杀对象
    // 2. 有血仇标记(vendetta)
    // 3. 态度攻击性(aggressive)
}
```

### 2.2 伤害系统 (feature/damage.c)

#### 生命体系

《侠客行》采用双层生命系统：

| 属性 | 说明 |
|-----|-----|
| `qi`（气） | 相当于生命值，战斗中首先消耗 |
| `eff_qi`（有效气） | 气上限，受伤会下降 |
| `max_qi`（最大气） | 角色基础气上限 |
| `jing`（精） | 精神力 |
| `eff_jing`（有效精） | 有效精神力 |
| `max_jing`（最大精） | 最大精神力 |
| `jingli`（精力） | 战斗/练习消耗的能量 |
| `neili`（内力） | 内功修为，用于加力和绝招 |
| `max_neili`（最大内力） | 内力上限 |

#### 伤害处理

```c
// 普通伤害（消耗）
int receive_damage(string type, int damage, mixed who);

// 伤势（降低上限）
int receive_wound(string type, int damage, mixed who);

// 恢复
int receive_heal(string type, int heal);

// 疗伤（恢复上限）
int receive_curing(string type, int heal);
```

#### 死亡与昏迷

```c
// 昏迷
void unconcious() {
    // 失去意识，无法行动
    // 清理所有敌人
    // 一段时间后自动苏醒
}

// 死亡
void die() {
    // 玩家死亡后进入幽灵状态
    // 移动到死亡房间
    // 清除所有状态
    // 记录死亡原因
    // 给击杀者奖励
    // 给死亡者惩罚
}
```

#### 自然恢复

```c
int heal_up() {
    // 战斗中：缓慢恢复
    // 非战斗中：快速恢复
    // 消耗食物和水
    // 恢复气、精、精力、内力
}
```

### 2.3 战斗仲裁 (adm/daemons/combatd.c)

`COMBAT_D`是战斗系统的核心仲裁者，负责所有战斗计算。

#### 战斗流程

```
fight() -> do_attack()
    ↓
选择招式 -> 计算攻击/防御 -> 闪避/招架判定
    ↓
命中? -> 是：计算伤害 -> 造成伤害 -> 报告状态
    ↓
    否：闪避/招架描述
```

#### 战斗威力计算 (skill_power)

这是核心战斗公式之一：

```c
varargs int skill_power(object ob, string skill, int usage) {
    int status, level, power, jingli_bonus;
    
    level = ob->query_skill(skill);  // 技能等级
    // 处理临时攻击/防御加成
    // 计算精力加成（精力充沛则更强）
    // 最终 = (技能等级三次方/3 + 战斗经验) × 属性系数
}
```

#### 攻击流程 (do_attack)

详细步骤：

1. **技能选择**
   - 根据武器/准备技能选择攻击方式
   - 可能映射到门派高级技能

2. **行动选择**
   ```c
   action = me->query("actions");
   // 获取当前可用招式
   ```

3. **命中判定**
   ```c
   ap = skill_power(攻击方, 攻击技能, SKILL_USAGE_ATTACK);
   dp = skill_power(防守方, "dodge", SKILL_USAGE_DEFENSE);
   
   // 概率：ap / (ap + dp) 命中
   if (random(ap + dp) < dp) {
       // 闪避成功
       // 显示闪避描述
       // 给闪避方少量经验
   }
   ```

4. **招架判定**
   ```c
   if (random(ap + pp) < pp) {
       // 招架成功
       // 显示招架描述
       // 给招架方少量经验
   }
   ```

5. **伤害计算**
   ```c
   // 基础伤害
   damage = me->query_temp("apply/damage");
   
   // 武器伤害加成
   // 技能伤害加成
   
   // 加力伤害（jiali）
   if (my["jiali"] && (my["neili"] > my["jiali"])) {
       // 消耗内力，增加伤害
   }
   
   // 技能特殊效果 (hit_ob)
   // 武器特殊效果
   // 加精（jiajin）效果
   
   // 战斗经验修正
   ```

6. **造成伤害**
   ```c
   damage = victim->receive_damage("qi", damage, me);
   
   // 有几率造成伤势（降低eff_qi)
   if (random(damage) > armor) {
       victim->receive_wound("qi", ...);
   }
   ```

7. **战斗描述与状态报告**
   ```c
   result = damage_msg(damage, action["damage_type"]);
   message_vision(result, me, victim);
   report_status(victim, wounded);
   ```

#### 伤害消息系统

根据伤害类型和伤害大小，有详细的描述：

| 伤害类型 | 典型技能 |
|---------|---------|
| 瘀伤 | 拳掌、钝器 |
| 割伤/劈伤/砍伤 | 刀、剑、斧 |

## 3. 状态系统

| 状态 | 文件 | 说明 |
|-----|------|-----|
| 蛇毒 | snake_poison | 持续掉血掉精 |
| 醉 | drunk | 可能用于醉拳 |

| 身孕 | pregnant | 女性特有 |

### 3.2 状态生命周期

以蛇毒为例：

```c
int update_condition(object me, int duration) {
    // 显示中毒痛苦描述
    // 根据中毒者状态显示不同严重程度
    
    // 造成伤害
    me->receive_damage("qi", duration/2, ...);
    me->receive_wound("jing", duration/3, ...);
    
    // 状态衰减
    if (duration > 5 + skill) {
        me->apply_condition("snake_poison", duration - 5 - skill/10);
    } else {
        // 状态结束
        return 0;
    }
    return 1;
}
```

### 3.3 状态施加与解除

```c
// 施加状态
apply_condition(string type, int duration);

// 检查状态
query_condition(string type);

// 清除状态
clear_condition();  // 清除全部

// 状态在死亡时全部清除
```

### 5.2 自动绝招系统

以少林派的 auto_perform.h 为例：

```c
int auto_perform() {
    object me = this_object();
    object target = me->select_opponent();
    object weapon = me->query_temp("weapon");
    
    // 非战斗时如果受伤就疗伤
    if (!me->is_fighting()) {
        if (me->query("eff_qi") < me->query("max_qi"))
            exert_function("heal");
        me->set("jiali", 100);
        return 1;
    }
    
    // 根据战斗经验比例决定是否使用绝招
    if (!random((int)me->query("combat_exp") * 2 / 
               (int)target->query("combat_exp") + 1))
        return 0;
    
    // 根据武器/技能选择合适的绝招
    if (objectp(weapon)) {
        switch (weapon->query("skill_type")) {
            case "whip": 
                return perform_action("whip.chanrao");
            case "club":
                // 醉棍特殊处理
                break;
            case "sword":
                return perform_action("sword.sanjue");
        }
    } else {
        // 根据准备技能选择绝招
        if (me->query_skill_prepared("finger") == "nianhua-zhi") {
            return perform_action("finger.fuxue");
        }
        // ...其他技能
    }
    return 0;
}
```

## 6. 优缺点分析

### 6.1 优点

1. **高度灵活性**：
   - 每个技能是独立LPC对象，可以随意修改
   - 可以随时添加新技能和新状态
   - 门派平衡调节简单

2. **丰富的战斗描述**：
   - 每个招式有独特的描述
   - 伤害程度有细致区分
   - 状态描述增加代入感

3. **完整的战斗系统**：
   - 攻击-闪避-招架三层判定
   - 双层生命系统（气 + 伤）
   - 内力系统（内力 + 精力）

4. **可扩展的AI系统**：
   - auto_perform提供了统一接口
   - 每个NPC可以有独特战斗逻辑
   - 支持阵法等复杂战斗

5. **源代码保留了演化痕迹**：
   - damage.c.bak 见证了系统的迭代

### 6.2 缺点

1. **战斗节奏较慢**：
   - 回合制战斗依赖心跳
   - 每次攻击都有大量计算和消息输出

2. **平衡性维护困难**：
   - 公式中硬编码的数值较多
   - 没有统一的数值平衡框架
   - 高级技能与低级技能差距太大

3. **没有现代战斗UI**：
   - 纯文本，视觉反馈不足
   - 缺乏战斗策略深度分析

4. **多人战斗机制较简单**：
   - 只有简单的多目标随机选择
   - 没有真正的团队协作机制

## 8. 总结

《侠客行》的武功与战斗系统是一个经典的MUD战斗系统，具有以下特点：

1. **完整的三层技能体系**：基础技能→门派技能→绝招
2. **丰富的战斗描述**：提升玩家代入感
3. **灵活的NPC AI**：支持各种战斗策略
4. **细致的状态系统**：中毒、受伤等多样状态
5. **成熟的死亡与奖惩**：提供完整的战斗循环

虽然是较早的设计，但系统架构清晰，功能完整，至今仍是研究MUD战斗系统的优秀范例。
