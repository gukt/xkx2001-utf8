# 侠客行 MUD 架构拆解说明书

## 项目简介

这是对经典中文 MUD 游戏《侠客行》的完整架构拆解。本系列文档深入分析了 MudOS 驱动下的 LPC 游戏系统，从驱动层到游戏逻辑层进行了全面解析，涵盖守护进程、命令系统、对象继承、世界构建、武功战斗、多人交互等核心模块。

本系列文档不仅是代码的分析，更是对 90 年代中文 MUD 设计思想的记录与传承。

## 术语表

### 驱动层术语

- **MudOS**：《侠客行》使用的 LPC 驱动程序，用 C 语言编写
- **LPC**：Lars Pensjö C，专门为 MUD 设计的编程语言
- **Driver**：驱动程序，负责 LPC 代码执行、网络连接、对象管理
- **Eval Cost**：评估成本，用于限制单个函数执行时间，防止死循环
- **Master Object**：Master 对象，驱动与 mudlib 的桥梁，处理连接、错误等

### 对象系统术语

- **Object**：对象，LPC 的基本单元
- **Blueprint**：蓝图对象，也叫 Master Object，用于克隆
- **Clone**：克隆对象，从蓝图克隆出来的实例
- **Inherit**：继承，LPC 的单继承机制
- **Feature**：特性，一种 Mixin 模式，通过继承多个功能模块实现组合
- **d_base**：数据存储系统，通过 `set()`/`query()` 操作的映射表
- **Swap**：交换，对象长时间不活动时被交换到磁盘

### 守护进程术语

- **Daemon**：守护进程，单例对象，提供全局服务
- **LOGIN_D**：登录守护进程，处理玩家认证与角色创建
- **COMMAND_D**：命令守护进程，负责命令查找与分发
- **COMBAT_D**：战斗守护进程，战斗仲裁与伤害计算
- **CHANNEL_D**：频道守护进程，管理聊天频道
- **CHAR_D**：角色守护进程，初始化与尸体生成
- **NATURE_D**：自然守护进程，管理时间与天气

### 命令系统术语

- **Verb**：命令动词
- **Path**：命令路径，按权限分级的命令目录
- **Alias**：别名，简化命令输入
- **Enable**：技能启用/映射

### 游戏世界术语

- **ROOM**：房间基类，游戏世界的基本容器
- **Exit**：出口，房间之间的连接
- **Area**：区域，一组相关房间的集合
- **NPC**：Non-Player Character，非玩家角色
- **Reset**：重置，房间定期刷新 NPC 和物品

### 战斗系统术语

- **Skill**：技能/武功
- **Force**：内功/内力
- **Perform**：绝招，技能的特殊招式
- **Exert**：内功功能
- **Qi**：气血/生命
- **Jing**：精神/内力
- **Jingli**：精力/体力
- **Shen**：正气值，影响正邪立场

### 数据流向图

```
玩家输入
  │
  ├─→ command_hook() ──→ find_command() ──→ 命令文件
  │         │
  │         └─→ 别名处理 (ALIAS_D)
  │
  ├─→ 命令执行 ──→ set()/query() ──→ dbase 数据更新
  │         │
  │         ├─→ NPC 对话/交易/任务 (ask/beg/steal)
  │         ├─→ 战斗动作 (hit/cast/perform)
  │         ├─→ 移动探索 (go/ride/drive)
  │         └─→ 社交交互 (say/tell/chat/team)
  │
  ├─→ 消息生成 ──→ message() ──→ 过滤 ──→ 目标对象接收
  │         │
  │         ├─→ 房间广播 (tell_room)
  │         ├─→ 频道分发 (CHANNEL_D)
  │         └─→ 跨 MUD 转发 (inetd/services)
  │
  ├─→ 心跳事件 (heart_beat)
  │         ├─→ NPC AI 行为 (random_move/chat/attack)
  │         ├─→ 战斗循环 (COMBAT_D)
  │         ├─→ 状态更新 (condition/force)
  │         ├─→ 自然恢复 (qi/jing/jingli)
  │         └─→ 环境事件 (NATURE_D: 时间/天气)
  │
  ├─→ 死亡判定 ──→ die() ──→ 尸体生成 ──→ 阴间传送

  │
  └─→ 存档保存 ──→ save_object() ──→ .o 文件
```
