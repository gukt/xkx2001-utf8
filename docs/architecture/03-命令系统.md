# 侠客行命令系统分析

## 一、概述

命令系统是侠客行 MUD 游戏的核心交互机制，负责处理玩家输入的所有指令。从简单的移动（`go`）、查看（`look`）到复杂的战斗（`kill`）、技能使用（`perform`），都通过命令系统进行解析和处理。

命令系统在整体架构中的位置请参考 [[01-架构总览]]。

## 二、命令解析流程

### 2.1 完整处理链

当玩家输入一条命令时，其处理流程如下：

```
网络输入
  ↓
驱动层 (Driver)
  ↓
用户对象 (User Object) - process_input()
  ↓
别名处理 (Alias) - ALIAS_D
  ↓
命令钩子 (Command Hook) - /feature/command.c
  ↓
命令查找 (Command Lookup) - COMMAND_D
  ↓
权限检查 (Permission Check) - SECURITY_D
  ↓
执行命令 (Execute Command) - call_other()
```

### 2.2 command.c 命令解析逻辑

命令解析的核心在 `/feature/command.c` 文件中，让我们深入分析其实现：

#### 2.2.1 主要函数结构

```c
// 命令钩子 - 所有命令的入口
nomask int command_hook(string arg)

// 查找命令
string find_command(string verb)

// 启用玩家命令系统
nomask void enable_player()

// 禁用玩家命令系统
nomask void disable_player(string type)
```

#### 2.2.2 command_hook 详细流程

`command_hook` 函数是命令处理的核心，执行以下步骤：

1. **获取命令动词**：使用 `query_verb()` 获取用户输入的命令部分
2. **去除前导空格**：调用 `remove_leading_space()` 清理命令
3. **尝试移动命令**：检查是否为方向，如果是且环境有该出口，调用 `go` 命令
4. **查找标准命令**：通过 `find_command()` 在命令路径中查找
5. **表情命令**：检查是否为表情（`emote`），由 `EMOTE_D` 处理
6. **频道命令**：检查是否为频道消息，由 `CHANNEL_D` 处理

```c
nomask int command_hook(string arg)
{
    string verb, file;

    // 获取命令动词
    verb = query_verb();
    if ((verb = remove_leading_space(verb)) == "")
        return 0;

    // 第一步：检查是否为方向移动
    if (!arg && environment() &&
        stringp(environment()->query("exits/" + verb)) &&
        stringp(file = find_command("go")) &&
        call_other(file, "main", this_object(), verb)) {
        // 成功执行 go 命令
    }
    // 第二步：查找并执行标准命令
    else if (stringp(file = find_command(verb)) &&
             call_other(file, "main", this_object(), arg)) {
        // 成功执行命令
    }
    // 第三步：尝试表情命令
    else if (EMOTE_D->do_emote(this_object(), verb, arg)) {
        // 成功执行表情
    }
    // 第四步：尝试频道命令
    else if (CHANNEL_D->do_channel(this_object(), verb, arg)) {
        // 成功发送频道消息
    }
    else {
        // 命令未找到
        return 0;
    }

    return 1;
}
```

### 2.3 COMMAND_D 命令查找机制

命令查找由 `/adm/daemons/commandd.c` 实现，主要功能包括：

#### 2.3.1 命令哈希缓存

COMMAND_D 使用映射（mapping）来缓存目录中的命令列表，避免每次都调用 `get_dir()`：

```c
mapping search = ([]);

void rehash(string dir)
{
    int i;
    string *cmds;

    if (dir[sizeof(dir)-1] != '/') dir += "/";
    cmds = get_dir(dir);
    i = sizeof(cmds);
    while (i--) {
        // 只保留 .c 文件
        if (!sscanf(cmds[i], "%s.c", cmds[i])) {
            if (i == 0) cmds = cmds[1..sizeof(cmds)];
            else if (i == sizeof(cmds)) cmds = cmds[0..<2];
            else cmds = cmds[0..i-1] + cmds[i+1..<1];
        }
    }
    if (sizeof(cmds)) search[dir] = cmds;
}
```

#### 2.3.2 find_command 查找算法

查找命令时按照路径顺序依次搜索，找到第一个匹配的命令后立即返回：

```c
string find_command(string verb, string *path)
{
    int i;
    string cmd;

    if (!pointerp(path)) return 0;

    i = sizeof(path);
    while (i--) {
        // 如果目录未哈希，先进行哈希
        if (undefinedp(search[path[i]])) rehash(path[i]);
        if (undefinedp(search[path[i]])) continue;

        // 在该目录中查找命令
        if (member_array(verb, search[path[i]]) != -1) {
            cmd = path[i] + verb;
            // 检查权限
            if (SECURITY_D->valid_cmd(cmd, this_player(), "cmd_file"))
                return cmd;
            else
                return 0;
        }
    }
    return 0;
}
```

### 2.4 命令路径与优先级

根据 `/include/command.h` 中定义的路径，不同权限等级的用户有不同的命令搜索路径：

| 等级 | 路径 |
|------|------|
| ADMIN_PATH | `/cmds/adm/` → `/cmds/arch/` → `/cmds/wiz/` → `/cmds/imm/` → `/cmds/usr/` → `/cmds/std/` → `/cmds/skill/` |
| ARC_PATH | `/cmds/arch/` → `/cmds/wiz/` → `/cmds/imm/` → `/cmds/usr/` → `/cmds/std/` → `/cmds/skill/` |
| WIZ_PATH | `/cmds/imm/` → `/cmds/usr/` → `/cmds/std/` → `/cmds/skill/` → `/cmds/wiz/` |
| APR_PATH | `/cmds/imm/` → `/cmds/usr/` → `/cmds/std/` → `/cmds/skill/` → `/cmds/wiz/` |
| IMM_PATH | `/cmds/imm/` → `/cmds/usr/` → `/cmds/std/` → `/cmds/skill/` |
| PLR_PATH | `/cmds/std/` → `/cmds/usr/` → `/cmds/skill/` |
| UNR_PATH (未注册) | `/cmds/usr/` → `/cmds/std/` |
| NPC_PATH | `/cmds/std/` → `/cmds/skill/` |

这种设计有两个优点：
1. **优先级控制**：高级权限用户可以覆盖低级权限的命令
2. **安全性**：命令按目录分组，权限检查更加清晰

### 2.5 命令路径设置

在 `enable_player()` 函数中，根据玩家的巫师等级设置相应的命令路径：

```c
nomask void enable_player()
{
    if (stringp(query("id"))) set_living_name(query("id"));
    else set_living_name(query("name"));

    delete_temp("disabled");
    enable_commands();

                enable_wizard();

            case "(arch)":
                set_path(ARC_PATH);
                enable_wizard();
                break;
            case "(wizard)":
                set_path(WIZ_PATH);
                enable_wizard();
                break;
            // ... 其他等级

                set_path(PLR_PATH);
        }
}
```

```c
string process_global_alias(string arg)
{
    string *word;

    if (arg[0] == '\'') return "say " + arg[1..strlen(arg)-1];

    word = explode(arg, " ");
    if (sizeof(word) && !undefinedp(global_alias[word[0]])) {
        word[0] = global_alias[word[0]];
        return implode(word, " ");
    }

    return arg;
}
```

##### look 命令深度分析

`look` 命令是最复杂的标准命令之一，让我们详细分析：

```c
int main(object me, string arg)
{
    object obj;
    int result;

    if (!arg)
        result = look_room(me, environment(me));  // 查看房间
    else if ((obj = present(arg, me)) || (obj = present(arg, environment(me)))) {
        if (obj->is_character())
            result = look_living(me, obj);         // 查看玩家/NPC
        else
            result = look_item(me, obj);           // 查看物品
    } else
        result = look_room_item(me, arg);          // 查看房间物品/出口

    return result;
}
```

**look_room - 查看房间**：
- 显示房间名称、描述
- 检查是否为室外，调用 `NATURE_D->outdoor_room_description()`
- 显示出口列表（排除关闭的门）
- 显示房间内的玩家和物品（排除不可见的）
- 处理坐骑显示

**look_living - 查看生物**：
- 显示生物的描述
- 如果是人类且有年龄，显示大致年龄
- 如果同门派且非自己，显示门派关系（师兄/师叔等）
- 如果是配偶，显示配偶关系
- 显示生命值状态
- 显示身上的物品（区分装备和非装备）
- 有机率触发 NPC 主动攻击（根据善恶值）

```c
int look_living(object me, object obj)
{
    string str, limb_status, pro;
    mixed *inv;
    mapping my_fam, fam;

    // 通知被查看者
    if ((me != obj) && (environment(me) == environment(obj))) {
        if (!wizardp(me) || (wizardp(me) && !wizardp(obj) && !me->query("env/invisibility")))
            message("vision", me->name() + "正盯著你看，不知道打些什么主意。\n", obj);
    }

    // 门派关系判断
    if (obj != me &&
        mapp(fam = obj->query("family")) &&
        environment(me) == environment(obj) &&
        mapp(my_fam = me->query("family")) &&
        fam["family_name"] == my_fam["family_name"]) {

        if (fam["generation"] == my_fam["generation"]) {
            // 同辈
            str += sprintf(pro + "是你的%s%s。\n",
                my_fam["master_id"] == fam["master_id"] ? "": "同门",
                my_fam["enter_time"] > fam["enter_time"] ? "师兄": "师弟");
        } else if (fam["generation"] < my_fam["generation"]) {
            // 长辈
            if (my_fam["master_id"] == obj->query("id"))
                str += pro + "是你的师父。\n";
            // ...
        } else {
            // 晚辈
            // ...
        }
    }

    // ... 其他显示
}
```

##### go 命令深度分析

`go` 命令处理移动逻辑，非常复杂：

```c
int main(object me, string arg)
{
    string dest, mout, min, dir, rided_name;
    object env, obj, rided;
    mapping exit;
    object *f_obs;

    // 检查是否可以移动
    if (me->over_encumbranced())
        return notify_fail("你的负荷过重，动弹不得。\n");
    if (me->is_busy())
        return notify_fail("你的动作还没有完成，不能移动。\n");
    if (me->query_temp("cannot_move"))
        return notify_fail(me->query_temp("cannot_move_msg") + "\n");

    // 战斗中逃跑处理
    if (me->is_fighting()) {
        f_obs = me->query_enemy();
        if (objectp(f_obs[0]))
            if (5 + random((int)me->query("dex")) <= random((int)f_obs[0]->query("dex"))) {
                me->start_busy(1);
                return notify_fail("你逃跑失败。\n");
            }
    }

    // 检查精力
    if (me->query("jingli") < me->query("max_jingli")/10)
        return notify_fail("你已经精疲力尽，动弹不得。\n");

    env = environment(me);
    if (!env) return notify_fail("你哪里也去不了。\n");

    // 检查出口
    if (!mapp(exit = env->query("exits")) || undefinedp(exit[arg])) {
        if (query_verb() == "go")
            return notify_fail("这个方向没有出路。\n");
        else
            return 0;
    }

    // 检查是否允许离开
    if (!env->valid_leave(me, arg)) return 0;

    // 获取目标房间
    if (!stringp(dest = me->query_temp("new_valid_dest")))
        dest = exit[arg];

    if (!(obj = load_object(dest)))
        return notify_fail("无法移动。\n");

    // 处理坐骑
    if (objectp(rided = me->query("rided")) &&
        environment(rided) == env && me->query("race") == "人类")
        rided_name = rided->name();
    else
        me->delete("rided");

    // 白天/夜间商店处理
    if (obj->query("day_shop") &&
        (day_event() == "event_night" || day_event() == "event_midnight"))
        return notify_fail(obj->query("short") + "晚上不开，请天亮了再来！\n");

    // ... 更多检查

    // 生成离开和进入消息
    if (me->is_fighting()) {
        // 战斗中的逃跑消息
        if (me->query("race") == "人类") {
            if (objectp(rided)) {
                mout = "伏在" + rided_name + "身上往" + dir + "落荒而逃了。\n";
                min = "骑着" + rided_name + "跌跌撞撞地跑了过来，模样有些狼狈。\n";
            } else {
                mout = "往" + dir + "落荒而逃了。\n";
                min = "跌跌撞撞地跑了过来，模样有些狼狈。\n";
            }
        } else {
            mout = replace_string(me->query("fleeout_message"), "$d", dir);
            min = me->query("fleein_message");
        }
    } else {
        // 正常移动消息
        // ...
    }

    // 消耗精力
    if (me->query("race") == "人类" && !me->query("env/invisibility")) {
        if (objectp(rided)) {
            if (random(5) == 0) me->add("jingli", -2);
            rided->add("jingli", -env->query("cost")*2);
            if (rided->query("jingli") <= 0) rided->unconcious();
        } else if (userp(me)) {
            me->add("jingli", -env->query("cost")*2);
            if (me->query("jingli") <= 0) me->unconcious();
        }
        message("vision", me->name() + mout, env, ({me}));
    }

    // 执行移动
    if (me->move(obj)) {
        me->remove_all_enemy();
        if (!objectp(me->query("rider")) && !me->query("env/invisibility"))
            message("vision", me->name() + min, environment(me), ({me}));
        me->set_temp("pending", 0);
        all_inventory(env)->follow_me(me, arg);
        return 1;
    }

    return 0;
}
```

##### get 命令深度分析

`get` 命令处理物品拾取：

```c
int main(object me, string arg)
{
    string from, item;
    object obj, *inv, env, obj2;
    int i, amount;

    if (!arg) return notify_fail("你要捡起什么东西？\n");
    if (me->is_busy())
        return notify_fail("你上一个动作还没有完成！\n");

    // 检查是否从容器中取
    if (sscanf(arg, "%s from %s", arg, from) == 2) {
        env = present(from, me);
        if (!env) env = present(from, environment(me));
        if (!env) return notify_fail("你找不到 " + from + " 这样东西。\n");
        if (wizardp(me) && living(env) && (wiz_level(me) <= wiz_level(env)) &&
            !env->query("ridable") && env->query_lord() != me)
            return notify_fail("你的巫师等级必须比对方高，才能搜身。\n");
    } else
        env = environment(me);

    if (!wizardp(me) && living(env) && env->query_lord() != me)
        return notify_fail("光天化日之下想抢劫啊？\n");

    // 检查是否指定数量
    if (sscanf(arg, "%d %s", amount, item) == 2) {
        if (!objectp(obj = present(item, env)))
            return notify_fail("这里没有这样东西。\n");
        if (!obj->query_amount())
            return notify_fail(obj->name() + "不能被分开拿走。\n");
        if (amount < 1)
            return notify_fail("东西的个数至少是一个。\n");
        if (amount > obj->query_amount())
            return notify_fail("这里没有那么多的" + obj->name() + "。\n");
        else if (amount == (int)obj->query_amount()) {
            return do_get(me, obj);
        } else {
            // 分裂物品
            obj2 = new(base_name(obj));
            obj2->set_amount(obj->query_amount() - amount);
            obj->set_amount(amount);
            if (me->is_fighting()) me->start_busy(3);
            if (do_get(me, obj)) {
                obj2->move(env);
                return 1;
            }
            obj2->move(env);
            return 0;
        }
    }

    // 全部捡起
    if (arg == "all") {
        if (me->is_fighting()) return notify_fail("你还在战斗中！只能一次拿一样。\n");
        if (!env->query_max_encumbrance()) return notify_fail("那不是容器。\n");

        inv = all_inventory(env);
        if (!sizeof(inv))
            return notify_fail("那里面没有任何东西。\n");

        for (i = 0; i < sizeof(inv); i++) {
            if (inv[i]->is_character() || inv[i]->query("no_get")) continue;
            do_get(me, inv[i]);
        }
        write("捡好了。\n");
        return 1;
    }

    // 捡单个物品
    if (!objectp(obj = present(arg, env)) || living(obj))
        return notify_fail("你附近没有这样东西。\n");

    if (obj->query("no_get"))
        return notify_fail("这个东西拿不起来。\n");

    return do_get(me, obj);
}

int do_get(object me, object obj)
{
    object old_env, *enemy;
    int equipped;
    string amt;

    if (!obj) return 0;
    old_env = environment(obj);

    if (obj->is_character()) {
        if (living(obj)) return 0;
    } else {
        if (obj->query("no_get")) return 0;
    }

    if (obj->query("equipped")) equipped = 1;
    if (obj->query_amount()) amt = chinese_number(obj->query_amount());

    // 移动物品
    if (obj->move(me)) {
        if (me->is_fighting()) me->start_busy(1);
        if (obj->is_character())
            message_vision("$N将$n扶了起来背在背上。\n", me, obj);
        else
            message_vision(sprintf("$N%s%s%s$n。\n",
                old_env == environment(me) ? "捡起" :
                (old_env->is_character() ?
                "从" + old_env->name() + "身上" + (equipped ? "除下" : "搜出") :
                "从" + old_env->name() + "中拿出"),
                obj->query_amount() ? amt : "一",
                obj->query_amount() ? obj->query("base_unit") : obj->query("unit")),
                me, obj);
        return 1;
    } else
        return 0;
}
```

#### 3.2.3 战斗类

```c
{
    string skill, teacher, master, skill_name, slow_msg;
    object ob;
    int master_skill, my_skill, gin_cost, slow_factor;
    int i, tmp, learn_times;

        return notify_fail("指令格式：learn|xue <某人> <技能> [次数]\n");

    if (sscanf(arg, "%s %s %d", teacher, skill, learn_times) != 3) {
        learn_times = 1;

            return notify_fail("指令格式：learn|xue <某人> <技能> [次数]\n");

    if (me->is_fighting())
        return notify_fail("临阵磨枪？来不及啦。\n");

    if (!(ob = present(teacher, environment(me))) || !ob->is_character())
        return notify_fail("你要向谁求教？\n");

        return notify_fail("你的潜能不够，没有办法再成长了。\n");

    for (i = 0; i < learn_times; i++) {
        if (!me->is_apprentice_of(ob) &&
            !(ob->recognize_apprentice(me)) &&
            !me->is_spouse_of(ob)) {
            return notify_fail(ob->name() + reject_msg[random(sizeof(reject_msg))]);
        }
    }

    // 峨嵋派男性学习缓慢
    if ((string)me->query("family/family_name") == "峨嵋派" &&
        (string)ob->query("family/family_name") == "峨嵋派" &&
        (string)me->query("gender") != "女性" &&
        (int)me->query_int() < 20 + random(25)) {
        slow_factor = 2;
        slow_msg = "想了良久，";
    } else {
        slow_factor = 1;
        slow_msg = "";
    }

    if (!master_skill = ob->query_skill(skill, 1))
        return notify_fail("这项技能你恐怕必须找别人学了。\n");

    notify_fail(ob->name() + "不愿意教你这项技能。\n");
    if (ob->prevent_learn(me, skill)) return 0;

    my_skill = me->query_skill(skill, 1);
    if (my_skill >= master_skill)
        return notify_fail("这项技能你的程度已经不输你师父了。\n");

    // 夫妻互相传授的限制
    if (me->is_spouse_of(ob)) {
        if (my_skill >= master_skill - 20 * (me->query("married_times") - 1))
            return notify_fail(ob->name() + "想到你和以前" +
                me->query("spouse/title") + "在一起的情形，有点不愿意教你这项技能。\n");
        else if ((ob->query("combat_exp") < 10000 ||
            me->query("combat_exp") < 10000) &&
            (string)SKILL_D(skill)->type() == "martial")
            return notify_fail("你们夫妻实战经验还不足，不能互相传授武艺！\n");
    }

    notify_fail("依你目前的能力，没有办法学习这种技能。\n");
    if (!SKILL_D(skill)->valid_learn(me)) return 0;

    gin_cost = 150 / (int)me->query("int");

    if (!my_skill) {
        gin_cost *= 2;
        me->set_skill(skill, 0);

        if (userp(ob)) ob->receive_damage("jing", learn_times * gin_cost / 5 + 1);
    } else {

        tell_object(ob, "但是你太累了，没有办法教" + me->name() + "。\n");
        return 1;

    gin_cost = learn_times * gin_cost * 3 / 2;

    // 学习消耗
    if ((int)me->query("jing") > gin_cost) {
        if ((string)SKILL_D(skill)->type() == "martial" &&
            my_skill * my_skill * my_skill / 10 > (int)me->query("combat_exp")) {
            printf("也许是缺乏实战经验，你对%s的回答总是无法领会。\n", ob->name());

            me->add("potential", -learn_times);

            tmp = 0;
            for (i = 0; i < learn_times; i++)

        write("你今天太累了，结果什么也没有学到。\n");
    }

    me->receive_damage("jing", gin_cost);

    return 1;
}
```

```c
        notify_fail("你所使用的外功中没有这种功能。\n");

        if (SKILL_D(skill)->perform_action(me, arg)) {

            return 1;
        } else if (SKILL_D(martial)->perform_action(me, arg)) {
            me->apply_condition("perform", martial);
            return 1;
        }
        return 0;
    } else if (sizeof(prepare) == 2 &&
        stringp(skill1 = me->query_skill_mapped(martial1)) &&
        stringp(skill2 = me->query_skill_mapped(martial2))) {
        // 组合技能处理
        notify_fail("你所使用的外功中没有这种功能。\n");

        if (SKILL_D(skill1)->perform_action(me, arg)) {
            me->apply_condition("perform", martial);
            return 1;

            me->apply_condition("perform", martial);
            return 1;
        } else if (SKILL_D(martial1)->perform_action(me, arg)) {
            me->apply_condition("perform", martial);
            return 1;
        } else if (SKILL_D(martial2)->perform_action(me, arg)) {
            me->apply_condition("perform", martial);
            return 1;
        }
        return 0;
    }

    return notify_fail("你请先用 enable 指令选择你要使用的外功。\n");
}
```

```c
int main(object me, string arg)
{
    string objname, func, param, euid;
    object obj;
    mixed *args, result;
    int i;

    if (arg) {
        if (sscanf(arg, "-%s %s", euid, arg) == 2) {
            // 设置 euid，仅 admin 可用
            if ((string)SECURITY_D->get_status(me) != "(admin)")
                return notify_fail("你不能设定自己的 euid。\n");
            seteuid(euid);
        } else
            seteuid(geteuid(this_player(1)));

        if (sscanf(arg, "%s->%s(%s)", objname, func, param) != 3)
            return notify_fail("指令格式：call <物件>-><函数>(<参数>, ...)\n");
    } else
        return notify_fail("指令格式：call <物件>-><函数>(<参数>, ...)\n");

    // 查找对象
    obj = present(objname, environment(me));
    if (!obj) obj = present(objname, me);
    if (!obj) obj = find_player(objname);
    if (!obj || !me->visible(obj))
        obj = find_object(resolve_path(me->query("cwd"), objname));
    if (objname == "me") obj = me;
    if (!obj) return notify_fail("找不到指定的物件。\n");

    // 防止滥用 call 命令操作其他玩家
    if (base_name(obj) == "/clone/user/user" ||
        base_name(obj) == "/u/zhuang/marriage/user") {
        if (obj != me) {
            if ((string)SECURITY_D->get_status(me) != "(admin)" &&
                (string)SECURITY_D->get_status(me) != "(arch)" &&
                (string)SECURITY_D->get_status(me) != "(wizard)" &&
                (string)SECURITY_D->get_status(me) != "(designer)" &&
                (string)SECURITY_D->get_status(me) != "(creator)" &&
                (string)SECURITY_D->get_status(me) != "(apprentice)")
                return notify_fail("你没有直接呼叫这个玩家的函数的权力。\n");

            log_file("static/CALL_PLAYER",
                sprintf("%s(%s) call %s(%s)->%s(%s) on %s\n",
                    me->name(1), geteuid(me), obj->name(1), geteuid(obj),
                    func, param, ctime(time())));
        }
    }

    // 解析参数
    args = explode(param, ",");
    for (i = 0; i < sizeof(args); i++) {
        parse_command(args[i], environment(me), "%s", args[i]);
        if (sscanf(args[i], "%d", args[i])) continue;
        if (sscanf(args[i], "\"%s\"", args[i])) continue;
    }

    args = ({func}) + args;

    // 调用函数
    result = call_other(obj, args);

    // 显示结果
    for (i = 1; i < sizeof(args); i++)
        args[i] = sprintf("%O", args[i]);
    printf("%O->%s(%s) = %O\n", obj, func,
        implode(args[1..sizeof(args)-1], ", "), result);
    return 1;
}
```

```c
int main(object me, string arg)
{
    object ob;
    string promotee, old_status, new_status, new_site;
    int my_level, ob_level, level;
    string uid, *wiz, wizlist;
    int i;

    if (me != this_player(1)) return 0;

    // 只有 admin 可以使用（且必须是特定的 id）
    if (wizhood(me) != "(admin)" && me->geteuid() != "npc") return 0;
    if (me->query("id") != "hxsd")
        return notify_fail("你没有这个权利\n");

    if (!arg)

    // 显示列表
    if (arg == "list") {
        write(SECURITY_D->query_wizlist());
        return 1;
    }

    // 解析参数
    if (sscanf(arg, "%s %s %s", promotee, new_status, new_site) != 3)
        if (sscanf(arg, "%s %s", promotee, new_status) != 2)
            return notify_fail("指令格式：promote <使用者> <等级> <Access IP>\n");

    if (wiz_level(new_status) <= 0 && new_status != "(player)")
        return notify_fail("没有这种等级。\n");

    // 查找目标玩家
    if (!objectp(ob = present(promotee, environment(me))))
        ob = find_player(promotee);

    if (ob && !userp(ob))
        return notify_fail("你只能改变使用者的权限。\n");

    if (!ob) {
        // 如果不在线，尝试读取存档
        seteuid(getuid());
        ob = new("/clone/test/dummy");
        ob->set("id", promotee);
        if (!ob->restore()) {
            notify_fail("没有这个玩家。\n");
        }
        seteuid(getuid());
    }

    old_status = wizhood(ob);

    seteuid(getuid());
    if (!(SECURITY_D->set_status(ob, new_status, new_site,
            me->query("name") + "(" + me->query("id") + ")")))
        return notify_fail("修改失败。\n");

    message_vision("$N将$n的权限从 " + old_status + " 改为 " + new_status +
        "，连线地址改为: " + new_site + " 。\n", me, ob);
    seteuid(getuid());
    ob->setup();

    return 1;
}
```

#### 4.2.2 valid_cmd 权限检查

```c
int valid_cmd(string file, mixed user, string func)
{
    string euid, status, *path, dir;
    int i;

    if (!objectp(user))
        error("TRUST_D->valid_cmd: Invalid argument type of user.\n");

    if (func != "cmd_file") return 0;

    if (!authorized_cmds) {
        // 默认配置
        authorized_cmds = allocate_mapping(10000);
        authorized_cmds["cmds/arch"] = "(admin),(arch)";
        authorized_cmds["cmds/wiz"] = "(admin),(arch),(wizard)";
    }

    euid = geteuid(user);
    if (!euid) return 0;
    status = get_status(user);

    if (euid == ROOT_UID) return 1;

    path = explode(file, "/");

    // 检查排除列表
    for (i = sizeof(path) - 1; i >= 0; i--) {
        dir = implode(path[0..i], "/");
        if (undefinedp(exclude_cmds[dir])) continue;
        if (member_array(euid, exclude_cmds[dir]) != -1) return 0;
        if (member_array(status, exclude_cmds[dir]) != -1) return 0;
    }

    // std、skill、usr 目录所有人都可以用
    for (i = sizeof(path) - 1; i >= 0; i--) {
        dir = implode(path[0..i], "/");
        if (dir == "cmds/std" || dir == "cmds/skill" || dir == "cmds/usr")
            return 1;

        if (undefinedp(authorized_cmds[dir])) continue;
        if (member_array(euid, authorized_cmds[dir]) != -1) return 1;
        if (member_array(status, authorized_cmds[dir]) != -1) return 1;
    }

    log_file("/static/CMD_LOG",
        sprintf("%s(%s) cmds attempt on %s failed.\n",
            euid, status, file));
    return 0;
}
```

### 4.3 权限检查流程

当执行命令时，权限检查流程如下：

## 六、优缺点分析

### 6.1 文件系统作为命令注册表的优点

1. **灵活性高**
   - 添加命令只需创建文件，无需修改注册表
   - 命令可以动态更新和重载
   - 易于按目录组织和分类

2. **权限控制清晰**
   - 按目录分配权限，简单直观
   - 权限继承路径顺序，高级覆盖低级

3. **开发便捷**
   - 每个命令独立文件，易于版本控制
   - 修改单个命令不影响其他命令
   - 可以单独测试和调试

4. **易于扩展**
   - 可以通过修改搜索路径添加新的命令目录
   - 支持命令覆盖和重载

### 6.2 文件系统作为命令注册表的缺点

1. **性能开销**
   - 需要文件系统 I/O 操作
   - 需要 `get_dir()` 扫描目录
   - 虽然有缓存机制，但首次加载仍有开销

2. **查找效率**
   - 按照路径顺序逐个目录查找
   - 最坏情况需要遍历所有目录

3. **缺乏元数据**
   - 命令的帮助信息、参数格式等需要在文件内部定义
   - 无法统一获取所有命令的概览

4. **错误处理复杂**
   - 命令文件语法错误会导致整个命令失效
   - 缺乏编译期检查

### 6.3 与现代游戏引擎命令系统的对比

| 特性 | 侠客行 MUD | 现代游戏引擎 |
|------|------------|------------|
| 注册方式 | 文件系统 | 代码宏/注解注册 |
| 参数解析 | 手动 sscanf | 自动类型转换 |
| 帮助系统 | 独立 help 函数 | 自动生成文档 |
| 权限系统 | 目录路径 | 标签/属性系统 |
| 重载机制 | 路径覆盖 | 函数重载/多态 |
| 性能优化 | 哈希缓存 | 预编译/内联 |
| 类型安全 | 弱类型 | 强类型检查 |

## 八、总结

侠客行的命令系统是一个设计精巧的文件系统驱动命令注册表，具有以下特点：

1. **分层路径搜索**：按权限等级提供不同的命令搜索路径
2. **动态加载机制**：命令文件即插即用，无需重启
3. **权限与目录绑定**：简单清晰的权限模型
4. **统一命令结构**：main() + help() 标准模式
5. **别名与快捷方式**：提升用户体验

这个系统虽然有一些性能上的权衡，但其灵活性和可扩展性在 MUD 类型的游戏中表现出色。它的设计思想甚至可以为现代游戏的命令系统、控制台系统提供参考。
