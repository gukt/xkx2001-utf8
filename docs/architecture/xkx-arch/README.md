# 侠客行 MUD 现代化重构架构方案

> 本方案由研究 Agents Team（36 个 Agent，四阶段：现状分析 / 模式研究 / 方案设计 / 对抗验证，约 200 万 tokens / 970 工具调用 / 25 分钟）产出，并经对抗验证修正承重级假设后整合而成。

## 重构目标

将经典 LPC MUD《侠客行》（8412 LPC 文件）重构为：
- **后端**：Python + WebSocket（asyncio + uvloop）
- **前端**：现代 Web（React + TypeScript）
- **部署**：分布式大规模，承载高并发
- **平台**：UGC 平台，DSL 配置剧情场景
- **创作**：Agent 协作创作多题材世界（武侠 / 大航海时代 / 书院 / 穿越 / 现代剧情）

## 核心立场

1. **诚实面对迁移面**：68771 处同步调用是年级工作量，不是月级。
2. **差分测试门禁**：无差分测试通过的子系统不进下一阶段。
3. **单进程起步**：不要为 50 用户的系统预先分布式化。
4. **原型先行**：Ray 起步验证，不在原型前承诺自研运行时。
5. **容量以实测定**：按 2000-3000 并发规划，而非假设 5000-8000。
6. **解决痛点优先**：所有选型经"是否解决痛点"检验，砍掉"显得先进"的过度工程。
