## 一、当前架构是否偏离基线：综合判定

### 判定结论：六条收缩硬边界无回潮，但存在三类偏离（置信度 medium-high）

**六条收缩硬边界**（不分布式/运维后置/单机 1000+100/纯 Python/内存 JSON 外部玩家前迁 PG/3 开放问题已裁决）：实测 runtime deps 实为 4 个（pydantic>=2/pyyaml>=6/argon2-cffi>=21/websockets>=12），`grep src/xkx` 无分布式/Rust/Go/Redis/PG/K8s 引入。**硬边界层面无回潮**。（置信度 high）

但存在以下三类偏离：

#### 偏离 1：04 基线文档与实现/裁决不同步（文档两张皮）

| 04 位置 | 04 文本 | 实际状态 | 证据 |
|---|---|---|---|
| L74/L236 | "工时承诺有数据支撑"/"抽样校准 50-100 调用点实测工时" | [ADR-0056](docs/adr/ADR-0056-abandon-effort-estimation-ai-batched-migration.md) 决策 1 已退役工时估算路径，改 AI 分批+边跑边记 AI 成本 | ADR-0056 L40-42 |
| L240 | 检查点 7"逐子系统行为等价验证" | [ADR-0061](docs/adr/ADR-0061-job-data-binary-source-equivalence.md) 决策 1 定义三档裁决（数据结构级/API 契约级必做，算法级接受推断），决策 4 L197-199 明确"后续遇同类子系统照此三档裁决"=通用模式 | ADR-0061 决策 1 + 决策 4 |
| L276 | "录制运行中 LPC 命令流为 golden trace \| 旧系统在当前 Linux 不可运行" | [ADR-0009](docs/adr/ADR-0009-original-driver-runnable.md) 决策 1 修正为"FluffOS 3.0 x86_64 macOS 可通过 Rosetta 2 运行"，04 理由列未更新 | ADR-0009 决策 1 |
| L226 | kill criteria 7"18 个月未达 M3 -> 冻结迁移" | M3 已通过归档（[stage-m3-done.md](docs/progress-archive/stage-m3-done.md) 2026-07-14），该 kill criteria 已不活跃但 04 未标注状态 | PROGRESS.md L51 |

**影响**：实现者可能按旧文本引入已退役路径（工时估算）或误判验收未达成。04§七避坑 1 自述"文档与约束两张皮是最需先修"在项目自身复现。（置信度 high）

#### 偏离 2：关键架构不变量"字面落地但语义有缺口"

[CLAUDE.md](CLAUDE.md) 列"do_attack 七步文本与副作用交织不可分离（不得先算后 apply）"为铁律。[resolve_attack.py](engine/src/xkx/combat/resolve_attack.py) 确实保留了 message/effect 交织顺序（Effect 账本按 side_effects order 交织），但纯函数+apply_effects 后置 apply 导致三类 LPC 即时副作用语义丢失：

- **(a) receive_damage 即时改 qi**：[combatd.c](adm/daemons/combatd.c) L678 receive_damage 减免后 damage 重新赋值，L704 读 `your['qi']` 是改后值影响后续 exp 判定；[resolve_attack.py](engine/src/xkx/combat/resolve_attack.py) L254 `rng.rand(victim.max_qi + victim.qi) < damage` 读快照原值（victim.qi 是 context 快照，未经 receive_damage 减免）。
- **(b) 中途停战**：combatd.c L744-759 victim qi*2<=max_qi 则 remove_enemy+winner_msg，resolve_attack 未实现。
- **(c) wound 概率简化**：[resolve_attack.py](engine/src/xkx/combat/resolve_attack.py) L229 `rng.chance(4)` 是固定 1/4，而 [golden trace](engine/tools/golden_trace/baseline/meta.json) lpc_formula_map 显示 LPC wound 有 4 种概率（空手 kill 1/4、武器 kill 1/2、空手非 kill 1/7、武器非 kill 1/4）。

**根因**：ADR-0023 决策 2 认为 ledger 保留交织顺序即合规，但 lpc-fidelity 指出 ledger 仅保顺序不保即时副作用语义。此分歧在 architecture（"已落地"）与 lpc-fidelity（"语义有缺口"）两视角间未交叉对质。（置信度 high：缺口确认存在；置信度 medium：是否为可接受保真收敛待 golden trace 评估）

#### 偏离 3：AI 迁移引入不可预测的架构偏离（N=2 方向相反）

- [ADR-0059](docs/adr/ADR-0059-bboard-subsystem-migration-scope.md) 决策 5：实施 agent 在 do_list 补 wizard_only/poster_family 门控=保守偏离（擅自安全增强），ADR 裁决移除。
- [ADR-0039](docs/adr/ADR-0039-combat-tick-alignment.md)：kill/fight 命令内同步循环绕过 tick=激进偏离，持续至 M3 收官后才纠正。

**两例偏离方向相反**（保守 vs 激进），说明 AI 偏离不可预测、无固定模式可筛，守门无法用模式预防只能逐批 review。ADR 级人工审查确实有效捕获并纠正了这两例，但守门是被动的（依赖人工 review 触发，非系统自动拦截），ADR-0039 违反持续数周才被发现（"违反窗口"）。（置信度 high：偏离实例真实；置信度 medium：是否系统性倾向 N=2 不足证明）

## 三、发现更好想法时的安全介入流程（决策树）

```text
冒出架构新想法/发现更好做法
    │
    ├─ Q1: 是否冲击六条收缩硬边界（分布式/Rust/Go/PG/K8s/分布式网关）？
    │   ├─ 是 → 停止。这是硬边界，不可违反，除非用户显式解除。
    │   └─ 否 → Q2
    │
    ├─ Q2: 是否冲击 CLAUDE.md 关键架构不变量（tick=1s/combat-only 确定性/
    │       do_attack 交织/PronounContext 三元组/JSON 原子写/themed 治理 fail-closed）？
    │   ├─ 是 → 必须先写 ADR，关联 [05](docs/xkx-arch/05-第三轮专家对抗复审报告.md) dissent。
    │   │       ADR 需论证：为何偏离不变量、偏离后的替代保证、影响范围。
    │   │       用户复审拍板后才可实施。不得在迁移时夹带。
    │   └─ 否 → Q3
    │
    ├─ Q3: 是否涉及保真决策（改变行为等价语义）？
    │   ├─ 是 → 先写 ADR，明确标注为"保真收敛"或"保真偏离"。
    │   │       对照 [03](docs/strategy-review/03-复刻意义与取舍战略.md) 保真度分级表：
    │   │       "必须"级保真项偏离需用户拍板；
    │   │       "应该/可选"级可 ADR 裁决后实施。
    │   └─ 否 → Q4
    │
    ├─ Q4: 是否涉及产品/安全语义决策（非纯技术实现）？
    │   ├─ 是 → 独立 ADR 关联 dissent，不得在迁移时夹带。
    │   │       ADR-0059 决策 5 先例：迁移职责是复现规格非修正规格。
    │   └─ 否 → Q5
    │
    ├─ Q5: 是否需要验证未知架构假设（非已知路径校准）？
    │   ├─ 是 → 先做 spike（原型验证）再决定是否纳入基线。
    │   │       spike 范围限定"架构层新想法"，非每个实现细节。
    │   │       验证通过 -> 写 ADR 纳入基线；验证失败 -> 丢弃，记录原因。
    │   └─ 否 → Q6
    │
    └─ Q6: 纯实现细节，不冲击基线/不变量/保真/产品语义。
        → 可直接实现，完成后补 ADR 记录（如有偏离则补偏离说明）。
```

### 决策树使用说明

- **"先 ADR/先验证/后补 ADR/夹带移除"四档触发阈值**：
  - 涉及硬边界/不变量/保真决策 -> **先 ADR**（用户复审拍板后才实施）
  - 涉及产品/安全语义 -> **先 ADR**（独立 ADR，不得夹带）
  - 涉及未知架构假设 -> **先验证**（spike 原型验证后再决定）
  - 纯实现细节 -> **后补 ADR**（如有偏离则补偏离说明）
  - 迁移时夹带非规格的门控/增强 -> **移除**（严格对齐 LPC，产品决策独立 ADR）

- **现有机制覆盖度**：[CLAUDE.md](CLAUDE.md) ADR 机制覆盖"冒想法冲击基线时如何处理"（写 ADR 决定是否纳入+关联 dissent）。04§一 7 条原则 + §五 10 条范围检查点 + §八阶段决策检查点是事前/阶段级规则。**缺的是"何时一个迁移期决策大到需先写 ADR/先验证、何时可直接实现后补 ADR"的明文阈值**，本决策树补齐此缺口。（置信度 medium）
