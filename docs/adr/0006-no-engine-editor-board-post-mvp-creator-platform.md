# 引擎不做原版编辑器/留言板；创作者平台与留言机制列入 post-MVP

用户确认：LPC「编辑器」实为文选归档（`editord.c`）+ 留言板逐行输入（`F_EDIT` / `bboard`），**新引擎不实现**。创作者侧长期形态是独立 **Web 一站式平台**（题材创作、验证、上架/下架、分成提现、数据统计、账户、AI Agent 辅助），与引擎进程分离——引擎只保留内容包加载/校验契约与运行时护栏。留言板/异步公告板在现代游戏中不常见，**推迟到多人世界互动阶段再评估**，不进 M0–M4。子系统 9「编辑器」由「现代化改造」**改判为丢弃**（二次改判；坐骑之后第二条）。详见 [.scratch/mvp-scope/post-mvp-backlog.md](../../.scratch/mvp-scope/post-mvp-backlog.md)。本 ADR 修正 [0005](0005-m3-ugc-loop-creation-surface.md) 中「编辑器维持现代化改造」的归类结论；0005 关于 M3 包外创作面的其余决策仍有效。

## 考虑过的选项

- **编辑器维持现代化改造**：否决。引擎侧既不做 LPC 文选/逐行编辑，未来创作 UX 也不落在引擎内，档位应反映「对引擎无设计参考价值」。
- **把 Web 创作者平台塞进 M3/M4**：否决。M3 是包→加载→可玩；M4 是商业化数据模型。平台是独立产品，排入 post-MVP 里程碑再排期。
- **MVP 做游戏内留言板**：否决。现代体验不优先；频道等实时交互已在 MVP 必做；异步留言等多人世界互动设计时再定。
