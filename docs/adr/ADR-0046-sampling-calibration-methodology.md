## 背景

[04](../xkx-arch/04-迁移路径与避坑清单.md) §三阶段 0 任务 6 定义："68771 调用点按难度分层抽 50-100 个实测工时"，验收"工时承诺有数据支撑"。[05](../xkx-arch/05-第三轮专家对抗复审报告.md) "加 20 项"明确要求"抽样校准硬交付物"。该任务一直后置（[08](../xkx-arch/08-阶段-0-实施计划.md) 标注"可后置可选项，非阶段 0 门禁"），M3 收官后作为可穿插推进项启动。

68771 = 全仓库 `->` call_other 同步调用（[00](../xkx-arch/00-愿景约束与总纲.md) §11 / [_archive/01](../xkx-arch/_archive/01-关键修正与避坑清单.md) 实测）。[_archive/01](../xkx-arch/_archive/01-关键修正与避坑清单.md) §373 标注 P0"精确统计迁移面（68771 调用点分类）"。

## 决策

1. **A/B 两阶段划分**：阶段 A（枚举 + 分布统计 + 抽样方案，轻量可验证）先行；阶段 B（手工实测 50-100 样本工时 + 全量推算）在 A 产出确认后启动。理由：收敛优先于完备（00 核心立场 4），A 的分布数据是 B 的前提，且 A 本身即是硬交付物（精确统计迁移面）。

2. **迁移单位推迟到数据驱动决策**：68771 个 `->` 大量是模式化调用（`obj->query("key")` 对应新引擎读组件，框架统一处理），按"调用点"实测会高估工作量。改用更粗单位（函数/文件/语义单元）需分布数据支撑。阶段 A 先按最细粒度（调用点）枚举，同时聚合到文件/函数级统计，报告给出"迁移单位"建议，阶段 B 启动前拍板。

3. **分层维度：方法名 + 子系统为主**：成本低、覆盖主要变异源。复杂度层级（纯数据读写 / 带逻辑 / 跨对象状态变更）作为附加分析，不作为主分层维度（分类成本高，阶段 A 不投入）。

4. **扫描范围**：聚焦 `->` call_other 形式（68771 主线指标）。`call_other()` efun 形式数量少，后置补充。

## 不做

- 阶段 A 不实测工时、不实现调用点迁移（阶段 B）
- 不扫描 `call_other()` efun 形式（后置）
- 不扫描 `.h` 文件（call_other 在 `.c` 实现）

## 后续

- 阶段 B 启动条件：阶段 A 报告确认 + 迁移单位决策
