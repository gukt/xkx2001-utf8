# 不做 LPC 行为等价验证

新目标(题材无关核心 MUD 引擎 + UGC 创作层 + 官方轻量武侠题材包 MVP)下,不再以"新实现与原版《侠客行》LPC 行为一致"作为约束或验证目标——无论是位等价(逐随机数序列对齐)、统计行为等价(伤害分布/概率一致),还是运行时对照(golden trace 录制回放)。LPC 源码与已归档的架构拆解文档,仅作设计灵感与术语参考,不再是"规格源"。

此决定推翻 [ADR-0009](../archive/adr/ADR-0009-original-driver-runnable.md) 确立的"行为等价验证分三层(规格自证 / 运行时对照 / 算法级推断)"框架——该框架服务于"全量复刻侠客行"这一旧目标,旧目标已在 2026-07-17 的项目重设中被放弃,验证框架随之失去存在意义。同理放弃 [ADR-0009](../archive/adr/ADR-0009-original-driver-runnable.md) 中"golden trace 作为辅助验证手段"的定位。

## 考虑过的选项

- **沿用三层验证框架,仅收窄范围到 MVP 涉及的子系统**:被否决。即便范围收窄,"跟原版对齐"这件事本身对新目标(题材无关引擎 + UGC)已无意义——新引擎要服务任意题材,不是《侠客行》的替代品。
- **保留统计行为等价作为引擎能力的一次性验证手段**(证明新引擎有能力精确复现一段历史设计,验证完即弃用):被否决,用户选择完全不需要这层验证,连一次性的验证也不做。

## 影响

- 专门服务于此目标的基建(golden trace、逐条对齐 LPC 随机数公式的战斗结算)已随工作区绿场清空移除；旧树见 tag `archive/engine-pre-m1-rewrite`，处置见 [ADR-0002](0002-engine-workspace-greenfield-reset.md) 与 [04 号票](../../.scratch/mvp-scope/issues/04-engine-code-disposition.md)。
- 36 个已发现子系统(见 [00-子系统发现报告](../archive/xkx-arch/_archive/_侠客行%20MUD%20架构拆解说明书/00-子系统发现报告.md))的归类依据,从"保真度分级"改为"对新引擎与 MVP 题材包的设计参考价值"。
