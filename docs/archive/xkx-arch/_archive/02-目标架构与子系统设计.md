# 目标架构与子系统设计

> 本文档是 10 个子系统设计方案的整合，已吸收 [01-关键修正与避坑清单](01-关键修正与避坑清单.md) 的修正。每个子系统含「设计理念 / 核心组件 / 关键接口 / 旧->新映射」。

---

## 总体架构分层

```
┌─ 接入层 ────────────────────────────────────────────────┐
│ WSGateway · AuthService · SessionManager · InputPipeline│  无状态·水平扩展
└────────────────────────────┬───────────────────────────┘
                             │ CommandMessage{verb,args,ctx,seq}
┌─ 游戏分片层 (WorldShard, 有状态) ──────────────────────┐
│ ActorRuntime(Ray) · ECS World(SparseSet) · TickSimulator│  按区域共同体分片
│ VerbRouter · CombatCoordinator · Region Supervisor       │
│ Systems: Combat/Skill/Condition/Heal/Equip/Movement/... │
└────────────────────────────┬───────────────────────────┘
                             │ 事件 / 投影 (tick 边界聚合)
┌─ 状态与服务层 ────────────────────────────────────────┐
│ PostgreSQL(权威状态+审计) · Redis(热状态·控制面) · NATS │
│ SchedulerService · ProjectionStore · ReferenceStore     │
└────────────────────────────┬───────────────────────────┘
                             │
┌─ UGC 与创作层 ────────────────────────────────────────┐
│ DSL编译器(4层) · CPK+provenance · WASM/RestrictedPython  │
│ Agent编排(LangGraph) · MCP验证 · ThemeRegistry           │
└──────────────────────────────────────────────────────────┘
```

**寻址统一**：`world://<shard>/<region>/<room>#<entity-uuid>` 替代 LPC `file_name`（解耦身份与代码路径，支持热重载与 UGC 多版本共存）。

## 子系统 1：网关与会话层

**理念**：把 MudOS 的 master.connect + login.c/logind.c 的 input_to 回调链 + F_MESSAGE + net_dead/reconnect，拆解为独立、无状态、可水平扩展的网关与会话层。

### 组件
- **WSGateway**：aiohttp+uvloop，SO_REUSEPORT 多 worker，单机数万连接。帧编解码（MsgPack/JSON）、origin 校验、per-IP/per-session 令牌桶限流、会话粘附路由、心跳 Ping/Pong 20s。
- **SessionManager**：session_id->{character_id, world, shard_node, state, last_seq, netdead} 状态机。状态流转 CONNECTING(300s 超时)->AUTHENTICATED->ACTIVE->SUSPENDED(900s 挂起窗口)->RESUMING->ACTIVE/CLOSED。外部化到 Redis。
- **AuthService**：argon2 替代 DES crypt()，JWT(RS256, access 5-15min + refresh 7d)。token 载 character_id/world/wiz_level。BAN_D（登录闸, banned_sites）+ REGBAN_D（建号闸, noreg_sites）双迁移。
- **ProtocolCodec**：统一帧 {type, version, seq, trace_id, payload}。type ∈ {snapshot, delta, event, ack, heartbeat, resume}。
- **ReconnectReconciler**：重连三步（校验 token+session -> 客户端上报 last_seq -> ring buffer 重放 delta，缺口超容量退化全量 snapshot_resend）。
- **TelnetAdapter**：过渡期兼容，以 receive_message(msgclass, msg) 为渲染缝。出站渲染 ANSI+代词替换，入站解析为 Command。

### 关键接口
- 下行 Frame: `{type:'snapshot'|'delta'|'event', seq, trace_id, payload}`（payload.subclass ∈ channel:/outdoor:/weather:/vision/tell_room/...）
- 重连: `ResumeFrame{session_id, last_seq}` -> `ReconnectResult{current_seq, replay_buffer|snapshot_resend}`
- REST: `POST /auth/login` -> `{access_token, refresh_token, character_id, world}`

### 旧->新映射（关键）
- `master.c::connect()` new(LOGIN_OB) -> WSGateway WS 连接回调创建 Session 记录（CONNECTING）
- `logind.c` input_to 串行回调链 -> AuthService POST /auth/login（异步单次）
- `user.c::net_dead()` set_heart_beat(0)+call_out(user_dump,900) -> SessionManager SUSPENDED + 挂起定时器（**netdead 停 tick + remove_all_enemy，沿用现状语义，删除"继续被仿真"表述**）
- `MAX_USERS=50`（实测）-> 弹性扩缩，按资源配额
- `feature/alias.c` cnt 洪水计数器 + 天雷惩罚 -> 网关 RateLimiter 令牌桶（**天雷惩罚单独建模为业务反作弊子系统**）
- `force_me` ROOT_UID 后门 -> 显式 PrivilegedAction API + 审计日志

## 子系统 2：分布式世界服务器

**理念**：基于虚拟 Actor 模型，**Ray 起步而非自研**。LPC 对象世界与 Actor 高度同构。

### 组件
- **WSGateway 网关层**：见子系统 1。
- **ShardSupervisor**：管理 WorldShard 生命周期。监督策略：玩家/房间 **one-for-one**，区域 **one-for-one + 持久化重建**（非 all-for-one）。
- **WorldShard**：有状态游戏节点。ActorRuntime + SimulationTick（1-2s）+ AOIManager（同房间实体集合 + 邻域静态元数据只读缓存，非 MMO 空间 AOI）。
- **PlacementService**：按 EntityID 寻址、按需激活、钝化/激活。**热路径用一致性哈希环 `shard=hash(entity_id)` 无外部依赖解析**，Redis 仅控制面。
- **MigrationCoordinator**：跨区 handoff 编排：freeze->snapshot(dbase+inventory+combat+team 写 Redis 附 TTL)->activate->replay->deactivate。组队批量 handoff 保持顺序。**每步定义超时/补偿/最终一致目标**。
- **GlobalStateService**：32 守护进程全量分类表。无状态副本（weapond/emoted/chinesed）/ 全局只读（natured 昼夜相位/commandd 注册表）/ 全局可变（channeld pub/sub 替代 users() 遍历、securityd、marryd）。
- **BoundaryRoomAgent**：分片边界房间跨分片事件代理，AOI 跨分片订阅转发。
- **CombatCoordinatorActor**：region/战斗级 Coordinator 串行裁决回合。同房间参与者同节点亲和。热门房间（>30 人）拆子 Coordinator 或简化回合模式。

### 关键接口
- `EntityAddr: 'world://<shard>/<region>/<room>#<entity-uuid>'`
- `ActorRef.send(msg, ctx)` / `ActorRef.ask(msg, ctx, timeout) -> Result`（ctx 携带调用者 addr + 能力令牌）
- `HandoffRequest{entity_id, target_shard, snapshot, team_members[], seq}`
- `CombatRoundResolved{combat_id, participants[], actions[], rng_seed, ts}`（内嵌逐击明细数组）

### 旧->新映射
- `file_name` 即身份又即路径 -> EntityID(UUID) 解耦
- `object` 指针引用 this_object()/environment()（3450 处环境，68771 处方法调用）-> ActorRef(addr) 仅传 addr 或值快照
- `call_other` 同步阻塞 -> ActorRef.ask 带超时（**68771 处是主线工作量，先迁移 go/move/combat/channel**）
- `master.c crash()` 全服宕 -> ShardSupervisor 分层监督 + CRASHES 审计
- `epilog/preload` 16 daemon -> 服务依赖描述 + 健康检查 + 32 daemon 分类表
- `room.c` 门状态 find_object 双面协调（对侧未加载静默失败）-> DoorState 共享存储键(Redis) + BoundaryRoomAgent

## 子系统 3：ECS 实体系统

**理念**：自研 SparseSet ECS（**单存储，不用 Archetype**）替换 LPC inherit 编译期静态混入。组件运行时动态挂载/卸载。

### 组件
- **ECS 运行时核心**：Entity 为 64 位整数 ID（高位 shard/region）。SparseSet 存储 O(1) 挂载/卸载/查询。
- **组件族**（按领域聚合）：Identity / Vitals(qi/jing/jingli+eff/max 三档) / Attributes / DerivedAttributes(dirty 缓存) / ModifierStack(presentation 覆盖型 + stat 累加型 + transform 数组栈，**非 ZSET**) / SkillSet / CombatState(enemy_id[] + ThreatTable) / EquipmentSlots / Conditions / Position / Inventory / Wallet / TeamRelation / ActionState / DynamicProps(UGC 动态键) / **Effect**(持续效果，一等公民) / **Progression**(combat_exp/potential/death_times)。
- **System 调度器**：CombatSystem/VitalsSystem/HealSystem/ConditionSystem/AttributeSystem/SkillSystem/EquipmentSystem/MovementSystem/ActionSystem/FinanceSystem/NPCAISystem。非均匀 tick。
- **EventBus**：替换 `this_object()->method()` 隐式耦合。**战斗热路径绕过总线**（resolve_attack 同步返回 CombatRoundResult，仅 tick 边界发聚合事件）。
- **Prefab/Archetype**：替代 default_ob 单级回退。支持多级继承。UGC 场景定义即 Prefab。
- **迁移适配层**：单进程 asyncio actor 内逐步将 LPC feature 行为迁入 Python System。

### 关键接口
- `create_entity(prefab_id, overrides) -> EntityID` / `query(*comp_types) -> SparseSetIter`（替代 users()/livings()/children()）
- `System.update(world, dt, ctx)` 纯函数
- `EffectInstance{effect_id, type, source_entity, target_entity, duration_ticks, mode: tick|wallclock, payload, stages, rng_seed, stacking_policy}`

### 旧->新映射
- inherit F_DBASE + dbase/tmp_dbase（**并行命名空间，非 base+diff**）-> 类型化 Component schema + DynamicProps
- default_ob 单级回退 -> Prefab 多级原型链
- F_ATTACK enemy/killer object 数组 -> CombatState.enemy_id[] + ThreatTable（ID 替代指针）
- receive_damage/receive_wound 直接 set dbase -> DamageEvent/HealEvent + VitalsSystem（**热路径同步**）
- query_str 等 apply 聚合 -> Attributes + ModifierStack(分三类) + DerivedAttributes dirty 缓存
- 472 个 SKILL_D 策略对象 -> SkillRef(skill_id, level) 组件 + SkillRegistry（技能是行为定义非数据）
- 72 个 condition daemon -> Conditions 组件 + ConditionHandler 协议（**on_tick 返回组合结构 continue/suppress_heal/expire**）
- `query_entire_dbase()` 活引用直改 -> **CombatSystem 迁移前必须先建字段->组件映射表 + Progression 组件**

## 子系统 4：命令与行为系统

**理念**：command_hook god-function 拆为显式中间件管线。`previous_object()` 隐式信任链显式化为 ActionContext + CapabilityToken。

### 组件
- **InputPipeline**：解码(UTF-8)/限流(Redis 令牌桶)/历史(!/!N + $1/$*)/别名展开/归一化。**Big5 转换是应用层中间件按 ctx.actor_id 查偏好，不下沉传输层**。
- **CommandRegistry**：命令自注册 Manifest(verb, aliases, category, args_schema, required_capabilities, cooldown, priority, help, handler_ref, cpk_id, version)。DB 持久化 + 版本标签原子切换（替代 rehash 无失效信号）。
- **VerbRouter**：显式有序解析器管线 ExitResolver(**保留 !arg 前置条件**)->CommandResolver->EmoteResolver->ChannelResolver。
- **ActionScheduler**：每 actor 可调度动作队列。immediate/delayed/repeating。Effect 可序列化随 Actor handoff 迁移。cancel/interrupt。
- **PermissionService**：按 (subject, action, resource) 求值。RBAC+ABAC 混合。exclude-before-trust。**fail-closed 默认** + 本地快照兜底。修复 authorized_cmds 拼写 bug。
- **CapabilityService**：previous_object -> 显式 CapabilityToken。危险动作需提升权限+审计。force_me 收敛为 PrivilegedAction API。
- **AliasService**：合并 aliasd global_alias + 个人 alias 为三层（系统/世界/用户）。
- **AuditService**：结构化审计 append-only。**安全关键审计先于提交同步写**。

### 关键接口
- `CommandManifest{verb, aliases, args_schema, required_capabilities, cooldown_ms, priority, help, handler_ref, cpk_id, version}`
- `async execute(ctx: ActionContext, args) -> ActionResult{status, message, effects, data}`
- `ActionScheduler.schedule(actor_id, effect, delay, repeat) -> action_id` / `cancel` / `interrupt` / `query_state`（state 含 suspended）
- `PermissionService.authorize(ctx, action, resource) -> Decision{allow, reason}`

### 旧->新映射
- COMMAND_D find_command + rehash -> CommandRegistry 自注册 + 版本标签
- command.h 8 套 *_PATH -> role_definition 继承链 + 能力集合
- command_hook god-function -> VerbRouter 显式有序解析器管线
- securityd valid_cmd 目录级 ACL -> PermissionService 单命令粒度（**fail-closed**）
- securityd authorized_cmds["cmd"] 拼写 bug -> 类型化 schema 启动期失败
- force_me ROOT_UID 后门 -> PrivilegedAction API + 审计
- 931 处 call_out -> ActionScheduler（**3570 字符串型机械转换 + 144 闭包型人工，Effect 内置 target 存活校验**）

## 子系统 5：战斗与玩法引擎

**理念**：保留 do_attack 七步管线 + AP/(AP+DP) 概率 + 三层资源 + 数据驱动招式的心智模型，用 ECS 替换 inherit/feature，用固定 tick 替换 heart_beat 上帝方法，用 Effect 替换 call_out 闭包。

### 组件
- **TickSimulator**：固定 tick（1-2s）。按子系统注册 tick 回调。保留 set_heart_beat(0/1) 按需启停。分级 tick 桶（tick0 战斗/恢复每 tick，tick1 condition 每 5-15 tick）。**持有 IdlePredicate 聚合判定 idle，ActivityChanged 事件驱动入队/出队**。
- **CombatSystem**：resolve_attack() 七步纯函数管线。读组件快照返回 CombatRoundResult，不直接 mutate。**CombatContext 快照边界**（战斗开始快照全体参与者+可观察环境）。seeded RNG。riposte/double_attack follow_up。
- **DamageSystem**：apply_damage/apply_wound/apply_heal/apply_cure。三层资源不变量 0<=current<=eff<=max。**热路径同步，Pydantic 仅边界校验**。
- **HealSystem**：战斗/非战斗不同恢复速率。魔法数字提取 HealRateConfig。
- **ConditionSystem**：ConditionStack + ConditionHandler 协议（**on_tick 返回组合 continue/suppress_heal/expire + 聚合规则**）。
- **EffectSystem**：**一等公民**。EffectInstance 可序列化可中断可迁移。stacking_policy(unique/refresh/stack/independent)。mode: tick|wallclock。崩溃恢复 clamp。
- **SkillSystem**：SkillDefinition(YAML Prefab) + SkillBehavior 注册表。技能是行为定义非数据。
- **DeathSystem**：DeathPolicy 策略对象（按题材可配）。free_rider 抢杀检测。
- **TargetingSystem**：ThreatTable 替代随机选择。MAX_OPPONENT=4 可配。
- **CombatRenderer**：文案抽离 damage_messages.json。代词替换纯函数。
- **DamageFormula**：skill_power 配置化表达式（level³/3 可改 level²/线性）。DamageFormulaConfig 可热重载。**面向 UGC 的公式进 WASM 或受限 AST**。

### 数据模型示例
```yaml
# kungfu/skill/yuenu-jian.skill.yaml
id: yuenu-jian
name: 越女剑法
type: sword
valid_enable: [sword, parry]
actions:
  - action: "$N施展一招「白露横江」，$w横削$n$l"
    force: 100
    dodge: 30
    damage: 20
    damage_type: 割伤
```

### 旧->新映射（关键）
- do_attack 直接读写 dbase（my['jingli']-=my['jiajin']）-> resolve_attack 纯函数读组件快照返回事件
- char.c heart_beat 上帝方法 -> TickSimulator 分系统独立 tick
- attack.c static object *enemy -> CombatState.enemy_id[] + ThreatTable（ID 替代指针）
- skill_power level³/3 -> DamageFormula 可配置表达式
- s_combatd.c 95% 重复副本 -> **废弃**，仅提取阵法合击/自创连击为 CombatModifier
- perform call_out 闭包递归 -> EffectSystem + Effect 多阶段 stages
- condition.c update_condition + CND_CONTINUE 位标志 -> ConditionSystem + **组合返回值**
- human.c 门派 if-else 加成 -> FamilyDefinition 门派数据声明 Prefab
- MudOS random() 全局 -> CombatRNG seeded（**+ CombatContext 快照，遗留内容行为等价非位等价**）

## 子系统 6：数据与状态层

**理念**：**权威状态库 + 选择性审计日志**（非全量事件溯源）。dbase(持久)+tmp_dbase(瞬时)分层保留。

### 组件
- **EventStore**：append-only 领域事件权威持久层。**仅高价值事件**（死亡/交易/婚姻/升级/门派变更）+ PvP 战报归档。战斗用回合聚合 CombatRoundResolved（**内嵌逐击明细**）。
- **SnapshotStore**：周期/关键节点全量快照。@persist 进快照，@transient 不进。
- **HotStateCache**：Redis。在线实体内存态。apply/ 修饰符存为 modifier 链表/栈（**非 ZSET**）。
- **EntityStateManager**：激活/钝化。load 最近快照 + 重放增量事件 = 当前状态。
- **SchemaRegistry**：pydantic v2 类型化组件。拼写错误编译期拦截。
- **ReferenceStore**：实体间关系存 ID 引用。**PostgreSQL 关系表（非 Neo4j）**。autoload path 数组 -> inventory_refs 懒加载。
- **SchedulerStore**：**APScheduler + Redis JobStore 单一后端**。**分层：短延迟(<30s)留内存，长周期走持久化**。保留 cancel 语义。
- **ProjectionStore**：CQRS 读侧。Wallet 账本双式记账替代 children() 扫描。排行榜/银行流水/战报独立部署。
- **WorldStateReconciler**：声明式 desired-state + reconcile（替代 reset 全量扫描）。
- **ArchiveMigrationService**：.o 存档迁移 + schema upcaster 链。

### 关键接口
- `CommandBus.dispatch(aggregate_id, command, ctx) -> Result`
- `EventStore.append(aggregate_id, events[], expected_seq) -> seq`（乐观并发）
- `SnapshotStore.save/load`
- `EntityStateManager.activate/passivate`

### 旧->新映射
- save_object .o 全量覆盖 -> SnapshotStore + EventStore（**非全量 ES**）
- F_DBASE dbase 无 schema -> 类型化 Component + SchemaRegistry
- tmp_dbase static 瞬时 -> Redis TTL + @transient
- default_ob 回退 -> Prefab 查询实例->模板回退链
- autoload path 数组 -> inventory_refs 懒加载
- object 指针引用 -> world:// 统一地址 + ReferenceStore（**PG 关系表非 Neo4j**）
- call_out（931 文件）-> SchedulerStore（**分层：短延迟留内存零开销取消**）
- heart_beat 2s 全局 -> 解耦 System 调度器
- reset 1800s 全量扫描 -> WorldStateReconciler 分片/按需 reconcile
- marryd 双方 dbase 双写无事务 -> ReferenceStore 关系事务原子提交
- **明确 Redis/PG 写入顺序协议与一致性强度**

## 子系统 9：前端与状态同步

**理念**：传输纯语义事件，渲染下沉前端。

### 组件
- **State Sync Engine**：**单一权威状态树**（整合 dbase 与 tmp_dbase 有效值），变更对比前后树生成 JSON-Patch ops + 单调 seq。**放弃 tmp_dbase=delta 映射话术**。
- **Projection Store**（前端）：Zustand/Jotai 细粒度响应式 + Immer。persistent 投影 + ephemeral 投影。
- **Message Render Layer**（前端）：代词三视角 render_message(template_key, actors) -> {to_me, to_you, to_room}。ANSI -> 主题样式 token。damage_msg -> 文案模板池(msg_key)。
- **Reconnect Buffer**：服务端 ring buffer(deque) 缓存 delta。**snapshot_resend 降级分支**。
- **Display Projection**（前端）：name.c short() 12+ 状态源聚合拆前端 DisplayName。

### 关键接口
- 下行 Snapshot: `{type:'snapshot', seq, state:{vitals, attrs, position, equipment, skills}}`
- 下行 Delta: `{type:'delta', seq, base_seq, ops:[{op:'replace',path:'/vitals/qi',value:80}]}`
- 下行 Event: `{type:'event', event:'combat_tick', payload:{attacker, victim, damage, msg_key, ctx}}`
- 代词渲染契约: `render_message(template_key, actors:{me, you, room}) -> {to_me, to_you, to_room}`

### 旧->新映射
- receive_message + ANSI + Big5 绑死 -> Render Adapter 渲染缝（Web 发语义事件，Telnet 拼 ANSI）

## 子系统 10：运维可观测与部署

**理念**：K8s + GitOps + OTel。有状态分片不能用无状态 HPA。

### 组件
- **Deploy Orchestrator**：Argo CD + Argo Rollouts + Helm。
- **Shard Lifecycle Manager**：region 亲和 + handoff 成本感知。先 Helm + 手动运维起步。
- **Metrics Collector**：prometheus_client。资源级 + 游戏语义级（tick p99、活跃实体、handoff 队列、call_out 积压）+ 业务级。
- **Structured Logger**：structlog JSON UTF-8 -> Vector -> **OpenSearch/ClickHouse**（非 Loki 做高维）。
- **Distributed Tracer**：OTel SDK + 自研 actor 信封 traceparent。**统一 span 策略**：边界 span 默认开，服务内用轻量子 span/span links，战斗关键步骤显式 span，长 trace 概率采样。**contextvars 传播助手 + lint 禁裸 create_task + trace 完整率指标**。
- **Health & Readiness**：**liveness 仅判事件循环死锁（极保守）**，readiness=依赖可达+tick 质量（摘流量不杀），startup 覆盖 20 万对象暖启动。加 PDB。
- **Config Center**：pydantic-settings + ConfigMap + NATS KV。**全局安全/数值配置两阶段 apply**，IP 封禁下发网关层。
- **Release Manager**：金丝雀（无状态边缘）+ 蓝绿（有状态分片）。**战斗变更灰度即全量**。
- **Canary Analyzer**：tick_p99/error_rate/handoff_fail_rate 对比。
- **Audit & Event Archive**：PG append-only 复用 ES 基础设施。**安全关键审计先于提交**。
- **Crash Handler**：OTel fatal + Alertmanager + K8s 自动重启 + 可选 RP 通知。
- **Scheduler Service**：NATS JetStream 持久化 + NATS KV 选主。**分层**（短延迟留内存）。
- **Autoscaler**：HPA（无状态）+ KEDA（**预测性+预留容量，非纯反应式**）。

- driver + portbind -> 容器化 WSGateway Deployment + Game Shard StatefulSet

## 跨子系统的一致性约束

1. **EntityID 全局唯一**：`world://<shard>/<region>/<room>#<uuid>`，替代所有 file_name。
2. **ActionContext 贯穿**：所有命令/系统调用携带 ctx（actor_id, session_id, world, shard, capability_token, trace）。
3. **事件 seq 单调**：每会话/每聚合域单调递增，支持断线重放。
4. **@persist/@transient 标注**：组件字段统一标注，决定 snapshot 行为。
5. **Effect 可序列化**：所有持续效果可跨分片迁移、可崩溃恢复。
6. **fail-closed 安全**：PermissionService Redis 不可达默认拒绝。
7. **差分测试门禁**：无差分测试通过的子系统不进下一阶段。
