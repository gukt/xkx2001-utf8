## 一、总览

### 目标

纯 Python 单进程 asyncio 跑通核心循环，验证两件事：

1. **语义正确性**：按 [08](08-阶段-0-实施计划.md) 9 层规格实现的核心循环行为等价于 LPC（go/move/combat/死亡轮回/NPC AI/登录）。
2. **性能达标**：单进程承载 1000 在线 + 100 并发，tick compute<100ms 非均匀 tick + 存档不阻塞事件循环（kill criteria 3 完整判定，[ADR-0012](../adr/ADR-0012-performance-microbenchmark.md) μs 基准的前置数据点已 GO）。

阶段 1 是 greenfield 重写的"从 stub 运行时到真实引擎"跃迁：阶段 -1 的 dict ECS + CLI REPL 升级为 SparseSet ECS + asyncio WS 服务器 + 8 段命令管线 + JSON 存档 + 6 ECS System。

## 三、任务分解与依赖排序

10 个里程碑分解为 10 个任务（T1-T10），按依赖排序。

### 依赖图

```text
T1 ECS 骨架升级 ──────┬──> T2 SchemaRegistry ──> T3 字段映射表
                      │
                      ├──> T5 JSON 存档（序列化组件）
                      │
                      ├──> T6 combat 确定性扩展 ──> T9 combat-sim
                      │
                      └──> T8 工具链（inspector）

T4 命令 8 段管线 ──────┬──> T7 WS 服务器 + 认证
                      │
                      └──> T8 工具链（profiler/replay）

T7 + T8 + T9 ─────────> T10 1000+100 集成测试（kill criteria 3）
```

### T1：ECS 骨架升级（M1-1）

- **目标**：dict ECS 升级为 SparseSet；加 Progression + Effect 一等公民组件；落定 ConditionHandler.on_tick 组合返回值契约。
- **输入**：[runtime/ecs.py](../../engine/src/xkx/runtime/ecs.py)（dict World）/ [runtime/components.py](../../engine/src/xkx/runtime/components.py)（11 组件）/ [spec/layer_b_object_base.py](../../engine/src/xkx/spec/layer_b_object_base.py)（F_DBASE 路径语义）/ [spec/layer_e_combat.py](../../engine/src/xkx/spec/layer_e_combat.py)（三层资源不变量）
- **产出**：
  - SparseSet 实现（entity_id -> 组件存储，按组件类型索引）
  - Progression 组件（exp/potential/skill_levels，LPC `query("combat_exp")` 等）
  - Effect 组件（一等公民：可序列化/可中断/可崩溃恢复，04 §三硬约束）
  - ConditionHandler.on_tick 组合返回值契约（副作用账本 + 状态变更，04 §三硬约束）
  - System 基类（tick 驱动的 update 方法）
- **依赖**：无（基础层）
- **验收**：SparseSet 查询性能不低于 dict（micro-benchmark 对比）；Progression/Effect 可序列化；ConditionHandler.on_tick 契约有属性测试
- **需 ADR**：SparseSet vs Archetype 选型 + Effect 一等公民设计 + ConditionHandler.on_tick 契约

### T2：SchemaRegistry 类型化组件（M1-2）

- **目标**：组件类型注册表，query 拼写错误启动期失败（非运行时静默 None）。
- **输入**：T1 升级后的组件 / [spec/layer_b_object_base.py](../../engine/src/xkx/spec/layer_b_object_base.py)（F_DBASE 路径访问语义）
- **产出**：SchemaRegistry（组件类型注册 + 字段 schema + 拼写检查）+ 启动期校验
- **依赖**：T1
- **验收**：`world.get(eid, Identity)` 拼写 `Identidy` 启动期失败；未知字段警告（衔接 [ADR-0008](../adr/ADR-0008-schema-validator-four-checks.md) SchemaValidator）
- **需 ADR**：SchemaRegistry 设计（与 DSL SchemaValidator 的边界）

### T3：字段->组件映射表（M1-3）

- **目标**：`query_entire_dbase` 调用点读写键集枚举完成（04 §三阶段 1 验收要求）。
- **输入**：LPC dbase 调用点扫描（`query`/`set`/`add_temp` 等）+ T2 SchemaRegistry
- **产出**：字段->组件映射表文档 + 代码常量（`DBASE_KEY_MAP`：LPC key -> 组件字段）
- **依赖**：T2
- **验收**：68771 调用点的核心键集（go/move/combat 路径）枚举完成；映射表覆盖 9 层规格涉及的 dbase 键
- **需 ADR**：无（T2 ADR 覆盖）

### T4：命令 8 段中间件管线（M1-7）

- **目标**：单段命令执行升级为 8 段中间件管线 + ActionContext + CapabilityToken；previous_object 显式化（155 处）。
- **输入**：[runtime/commands.py](../../engine/src/xkx/runtime/commands.py)（10 命令）/ [spec/layer_c_command.py](../../engine/src/xkx/spec/layer_c_command.py)（command_hook 四分支 + 18 方向别名 + find_command 逆序）/ [spec/layer_h_daemons.py](../../engine/src/xkx/spec/layer_h_daemons.py)（SECURITY_D valid_cmd fail-closed）/ [ADR-0014](../adr/ADR-0014-daemon-responsibility-redesign.md)（securityd -> PermissionService + CapabilityToken）
- **产出**：
  - 8 段中间件（草案：0 别名解析 / 1 权限校验 CapabilityToken / 2 命令查找 / 3 方向快捷 / 4 参数解析 / 5 previous_object 注入 / 6 执行 / 7 审计日志，具体分段 ADR 定）
  - ActionContext（actor/viewer/target 三元组，PronounContext 不变量，[CLAUDE.md](../../CLAUDE.md) 关键不变量）
  - CapabilityToken（不可伪造，02 裁决 HS256/内存 session token + 内存吊销集合）
  - previous_object 显式化（LPC `this_player()` / `previous_object()` 155 处映射）
  - force_me = PrivilegedAction（ROOT 门控 + 强制审计，[CLAUDE.md](../../CLAUDE.md) 不变量）
- **依赖**：T1（ECS）+ T2（SchemaRegistry）
- **验收**：go/kill/ask/give/quest 等 10 命令走 8 段管线行为等价；权限校验 fail-closed；PronounContext 三元组携带 viewer
- **需 ADR**：8 段管线分段 + ActionContext 设计 + CapabilityToken 实现 + previous_object 映射策略 + force_me 审计

### T5：内存权威 + JSON 存档（M1-5）

- **目标**：内存权威 + 本地 JSON 定时存档，原子写 + offload + dirty-flag 分摊 + 崩溃冷重启从 checkpoint 恢复。
- **输入**：[spec/layer_i_character.py](../../engine/src/xkx/spec/layer_i_character.py)（save 三步交织 + JSON 存档崩溃安全要求）/ [spec/layer_a_driver.py](../../engine/src/xkx/spec/layer_a_driver.py)（save_object 副作用）/ [CLAUDE.md](../../CLAUDE.md) 不变量（write-temp+os.replace 原子写 + 事件循环外 offload + dirty-flag 分摊）
- **产出**：
  - StorageBackend 策略接口（内存权威 + JSON 存档，PG 后置 kill criteria 8）
  - 原子写（write-temp + os.replace，不得重蹈 LPC `save_object` 全量覆盖无原子写覆辙）
  - offload 到事件循环外（`run_in_executor`，存档不阻塞 tick）
  - dirty-flag 分摊（只存脏实体，非全量）
  - 崩溃冷重启从 checkpoint 恢复
  - Effect 可崩溃恢复（与 T1 Effect 组件衔接，存档包含未完成 Effect 状态）
- **依赖**：T1（组件序列化）
- **验收**：原子写通过崩溃测试（写入中断不损坏）；存档不阻塞事件循环（tick profiler 验证）；冷重启恢复玩家状态
- **需 ADR**：JSON 存档格式 + 崩溃恢复协议 + dirty-flag 策略 + Effect 序列化

### T6：combat 确定性扩展（M1-6）

- **目标**：resolve_attack 纯函数扩展为单 tick 快照 + input log 可重放（combat 范围）；补全 ADR-0002 简化台账。
- **输入**：[combat/resolve_attack.py](../../engine/src/xkx/combat/resolve_attack.py) / [combat/conformance.py](../../engine/src/xkx/combat/conformance.py) / [spec/layer_e_combat.py](../../engine/src/xkx/spec/layer_e_combat.py)（do_attack 七步 + 31 处 random + 三层资源不变量）/ [ADR-0002](../adr/ADR-0002-resolve-attack-extraction.md) 简化台账
- **产出**：
  - CombatSystem（tick 驱动，调 resolve_attack + apply_effects）
  - 单 tick 快照（CombatContext + 全局状态快照）
  - input log（战斗内输入按序记录，可重放）
  - combat 确定性重放（同 seed + 同快照 + 同 input log -> 同输出，PYTHONHASHSEED=0 已验证 ADR-0012）
  - 补全简化台账：hit_ob/hit_by mapping / riposte 递归 / 武器类型完整集 / skill_power 公式 / combat_exp 防御折减
- **依赖**：T1（Effect 组件 + System 基类）
- **验收**：combat 范围确定性重放通过；ConformanceChecker 8 项检查通过；ADR-0002 简化台账 6 项补全
- **需 ADR**：combat 确定性边界（combat-only，全仿真后置 M3）+ 简化台账补全策略

### T7：单进程 WS 服务器 + 认证 + 重连（M1-4）

- **目标**：asyncio WS 服务器承载会话 + argon2 密码 + HS256 session token + 断线重连（ring 重放 / snapshot 降级）。
- **输入**：[cli.py](../../engine/src/xkx/cli.py)（命令解析，迁到 WS 入口）/ [spec/layer_h_daemons.py](../../engine/src/xkx/spec/layer_h_daemons.py)（LOGIN_D 13 阶段状态机）/ [spec/layer_i_character.py](../../engine/src/xkx/spec/layer_i_character.py)（visible 三级 + PronounContext viewer）/ [ADR-0014](../adr/ADR-0014-daemon-responsibility-redesign.md)（logind -> ConnectionSystem + AccountService）
- **产出**：
  - asyncio WS 服务器（stdlib asyncio，不用 uvloop，04 §六）
  - LOGIN_D 13 阶段状态机（logon -> get_id -> get_passwd -> make_body -> enter_world）
  - AccountService（argon2 密码哈希替换 LPC crypt()，ADR-0014 决策 7）
  - HS256 session token + 内存吊销集合（02 裁决，不引入 JWT RS256 + Redis 黑名单）
  - 断线重连：ring buffer 重放近期事件 / snapshot 降级（ring 不足时发全量快照）
  - ConnectionSystem（ADR-0014 的第 6 个 System）
- **依赖**：T4（命令管线，WS 收命令 -> 管线执行）+ T5（玩家状态存档）
- **验收**：单进程承载会话；断线重连 ring 重放 / snapshot 降级切换正确；argon2 密码验证通过
- **需 ADR**：WS 协议 + 重连策略（ring vs snapshot 切换条件）+ AccountService 设计

### T8：引擎工具链实现（M1-8）

- **目标**：实现 ADR-0013 三件工具（entity inspector / tick profiler / combat replay viewer），进程内模块。

### 关键不变量（CLAUDE.md，实施中不可违反）

- **tick=1s + compute<100ms + 非均匀 tick**，不得引入 50ms/20Hz 框架
- **Command 仅覆盖外部意图**，System tick 派生变更不经 Command；force_me=PrivilegedAction（ROOT 门控 + 强制审计）
- **combat 确定性范围=combat-only**，全仿真后置 M3
- **do_attack 七步文本与副作用交织不可分离**，不得"先算后 apply"
- **PronounContext 必须携带 viewer**（三元组 speaker/viewer/target）
- **JSON 存档崩溃安全**：write-temp+os.replace 原子写 + 事件循环外 offload + dirty-flag 分摊
- **存储接口以"持久化边界"抽象**（persist=崩溃恢复级耐久），非"save=权威写"
- **themed 治理（天雷/阴间/vote/法院）是平台级 fail-closed Python**，不落入 UGC
