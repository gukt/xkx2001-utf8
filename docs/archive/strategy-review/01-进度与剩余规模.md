## 三、推进变慢根因诊断

### 根因 1：推进方式经历调整（工时估算 -> 放弃 -> AI 分批）

[ADR-0048](docs/adr/ADR-0048-workload-estimation-by-call-point-enumeration.md)（07-14 枚举调用点 59270 + 80 样本）-> [ADR-0056](docs/adr/ADR-0056-abandon-effort-estimation-ai-batched-migration.md)（07-15 pilot 13 样本 30min/851k token -> 07-15/16 退役工时估算）。这不是"推进变慢"，而是"推进方式被介入调整"的过程性停顿。[ADR-0056](docs/adr/ADR-0056-abandon-effort-estimation-ai-batched-migration.md) 背景第 3 点原文确认"ADR-0048 决策 7 已否决 LLM 辅助纳入工时承诺但仍预设需回答人工工时"的前提被推翻。（置信度 high）

### 根因 2：分批策略有架构依据但缺内容迁移里程碑

[ADR-0056](docs/adr/ADR-0056-abandon-effort-estimation-ai-batched-migration.md) 决策 4"第一批=补架构缺口层"（item-as-entity/message facade/per-object save），决策 5"后续批=按子系统/缺口类型"。wear/wield 是 2.3 Equipment 命令层补强，属架构依赖前置。**真实缺口不是"挑软柿子"，而是分批策略有架构依据但缺内容迁移里程碑/批次/保真度分层**--6414 房间内容何时启动、如何分批、保真度如何分层，均未定义。（置信度 high）

### 根因 3：全量迁移无活跃止损线

[04](docs/xkx-arch/04-迁移路径与避坑清单.md) §四 kill criteria 7"18 个月未达 M3 -> 冻结迁移"针对 M3 线，M3 已通过归档（[stage-m3-done.md](docs/progress-archive/stage-m3-done.md) 2026-07-14），该 kill criteria 已不活跃。但 04 后置阶段表（L200-213）11 项全部触发条件驱动（分布式需 80% 承载/PG 需外部玩家测试/K8s 需 2+ 服务），**"全量迁移完成"无触发条件也无止损线**。迁移可能无限期延展无止损，这是治理盲区。（置信度 medium-high）

### 根因 4：AI 成本记录回路断档

[ADR-0056](docs/adr/ADR-0056-abandon-effort-estimation-ai-batched-migration.md) 决策 5 承诺"AI 成本（token/运行时间）边跑边记，替代人工工时作为规模参考"，但后续 5 批迁移（ADR-0057~0064）grep 零 AI 成本记录。[PROGRESS.md](PROGRESS.md) L46 列为 Next Up 待办至今未落地。退役工时估算后承诺的替代度量未建立执行回路，形成"决策采纳-度量断档"。（置信度 high）

## 四、ADR-0056 AI 分批评估

### pilot 实证

[ADR-0056](docs/adr/ADR-0056-abandon-effort-estimation-ai-batched-migration.md) pilot：13 样本（11 个 agent 并行迁移），30 分钟完成 11 个样本，851k tokens。这证明了 AI agent 分批迁移的技术可行性。

### 后续批次验证

| 批次 | ADR | 产出 | tests 增量 | AI 成本记录 |
|---|---|---|---|---|
| bboard | [ADR-0059](docs/adr/ADR-0059-bboard-subsystem-migration-scope.md) | bboard 子系统 | 含在 2312 | 无 |
| job_data | [ADR-0061](docs/adr/ADR-0061-job-data-binary-source-equivalence.md) | job_data + 12 API + 4 命令 | +50 (2371) | 无 |
| 武器 CPK 接线 | [ADR-0062](docs/adr/ADR-0062-weapon-cpk-wiring-postpone.md) | 17 数据层 CPK + 149 武器 | +4 (2387) | 无 |
| wield 命令批 | [ADR-0063](docs/adr/ADR-0063-wield-command-batch.md) | wield/unwield + WeaponDef 删除 | +16-6 (2397) | 无 |
| wear 命令批 | [ADR-0064](docs/adr/ADR-0064-wear-command-batch.md) | wear/remove + 145 护甲 | +19 (2416) | 无 |

**结论**：5 批迁移 tests 全绿证明推进方式适配当前阶段，但 AI 成本记录断档使规模参考真空无法填补。（置信度 high）
